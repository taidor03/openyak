# OpenYak 二次开发实战

## 概述

本文档通过实际案例演示如何对 OpenYak 进行二次开发。每个案例包含完整的需求分析、代码实现和测试步骤。

## 实战案例 1：添加自定义工具

### 需求

添加一个天气查询工具，让 Agent 可以查询指定城市的天气。

### 实现步骤

#### 1. 创建工具类

创建 `backend/app/tool/builtin/weather.py`：

```python
"""天气查询工具 - 使用 Open-Meteo 免费API"""

import logging
from app.tool.base import ToolDefinition, ToolResult
from app.tool.context import ToolContext
from app.tool.truncation import truncate_output
import httpx

logger = logging.getLogger(__name__)

WEATHER_CODES = {
    0: "晴朗", 1: "大部晴朗", 2: "多云", 3: "阴天",
    45: "雾", 48: "雾凇",
    51: "毛毛雨", 53: "中度毛毛雨", 55: "密集毛毛雨",
    61: "小雨", 63: "中雨", 65: "大雨",
    80: "阵雨", 95: "雷雨",
}

class WeatherTool(ToolDefinition):
    @property
    def id(self) -> str:
        return "weather"

    @property
    def description(self) -> str:
        return "查询指定城市的当前天气信息"

    @property
    def is_concurrency_safe(self) -> bool:
        return True  # 只读工具

    def parameters_schema(self) -> dict:
        return {
            "type": "object",
            "properties": {
                "city": {
                    "type": "string",
                    "description": "城市名称（中文或英文）"
                },
                "units": {
                    "type": "string",
                    "enum": ["metric", "imperial"],
                    "description": "单位：metric（摄氏度）或 imperial（华氏度）",
                    "default": "metric"
                }
            },
            "required": ["city"]
        }

    async def execute(self, args: dict, ctx: ToolContext) -> ToolResult:
        try:
            city = args["city"]
            units = args.get("units", "metric")

            async with httpx.AsyncClient(timeout=30.0) as client:
                # 获取城市坐标
                geo_resp = await client.get(
                    "https://geocoding-api.open-meteo.com/v1/search",
                    params={"name": city, "count": 1}
                )
                geo_data = geo_resp.json()

                if not geo_data.get("results"):
                    return ToolResult(error=f"未找到城市：{city}")

                loc = geo_data["results"][0]
                lat, lon = loc["latitude"], loc["longitude"]

                # 获取天气
                weather_resp = await client.get(
                    "https://api.open-meteo.com/v1/forecast",
                    params={
                        "latitude": lat, "longitude": lon,
                        "current": "temperature_2m,weather_code,wind_speed_10m",
                        "temperature_unit": units
                    }
                )
                weather = weather_resp.json()["current"]

                desc = WEATHER_CODES.get(weather["weather_code"], "未知")
                unit_sym = "°C" if units == "metric" else "°F"

                output = (
                    f"{city} 当前天气：\n"
                    f"- 温度：{weather['temperature_2m']}{unit_sym}\n"
                    f"- 天气：{desc}\n"
                    f"- 风速：{weather['wind_speed_10m']} km/h\n"
                    f"- 位置：{loc['name']}, {loc.get('country', '')}"
                )

                return ToolResult(
                    output=truncate_output(output),
                    metadata={
                        "city": city,
                        "temperature": weather["temperature_2m"],
                        "weather": desc,
                    }
                )
        except Exception as e:
            logger.error(f"Weather query failed: {e}", exc_info=True)
            return ToolResult(error=f"查询天气失败：{e}")
```

#### 2. 注册工具

在 `backend/app/tool/builtin/__init__.py` 中添加：

```python
from .weather import WeatherTool

def register_builtin_tools(registry: ToolRegistry) -> None:
    # ... 现有工具
    registry.register(WeatherTool())
```

#### 3. 配置权限

build agent 默认允许所有工具（`tools=[]`），天气工具会自动可用。如需显式控制，添加权限规则。

#### 4. 测试

```bash
# 启动后端
cd backend && uvicorn app.main:create_app --factory --reload

# 在对话中测试
# 用户: "帮我查询北京的天气"
# Agent 会调用 weather 工具
```

## 实战案例 2：添加自定义 Agent

### 需求

创建一个代码审查 Agent，只读代码并提供审查意见。

### 实现步骤

#### 1. 定义 Agent

在 `backend/app/agent/agent.py` 的 `BUILTIN_AGENTS` 中添加：

```python
"code_review": AgentInfo(
    name="code_review",
    description="代码审查专家，专注于代码质量、安全性和最佳实践",
    mode="primary",
    tools=["read", "glob", "grep", "search", "skill"],
    permissions=Ruleset(rules=[
        PermissionRule(action="deny", permission="*"),
        PermissionRule(action="allow", permission="read"),
        PermissionRule(action="allow", permission="glob"),
        PermissionRule(action="allow", permission="grep"),
        PermissionRule(action="allow", permission="search"),
        PermissionRule(action="allow", permission="skill"),
    ]),
    system_prompt=_load_prompt("code_review"),
),
```

#### 2. 创建 System Prompt

在 `backend/app/agent/prompts/` 创建 `code_review.txt`：

```
你是一个代码审查专家，专注于代码质量、安全性和最佳实践。

你的职责：
1. 审查代码的语法和逻辑错误
2. 检查潜在的安全漏洞
3. 评估代码的可读性和可维护性
4. 提供改进建议

审查要点：
- 变量命名是否清晰
- 函数是否单一职责
- 是否有重复代码
- 错误处理是否完善
- 是否遵循语言最佳实践

注意：你只能读取代码，不能修改。
```

#### 3. 前端自动发现

Primary 模式的 agent 会通过 `GET /api/agents` 自动出现在前端选择器中，无需额外前端代码。

## 实战案例 3：添加自定义 Provider

### 需求

集成一个自建的 LLM 服务（兼容 OpenAI API 格式）。

### 实现步骤

#### 1. 添加目录条目

在 `backend/app/provider/catalog.py` 中添加：

```python
PROVIDER_CATALOG["my_llm"] = {
    "id": "my_llm",
    "name": "My LLM Service",
    "base_url": "https://llm.mycompany.com/v1",
    "models": ["my-model-v1", "my-model-v2"],
    "env_key": "OPENYAK_MY_LLM_API_KEY",
}
```

#### 2. 无需创建新 Provider 类

因为自建服务兼容 OpenAI API 格式，直接使用 `GenericOpenAIProvider` 即可。在 `factory.py` 中添加路由：

```python
elif provider_id == "my_llm":
    return GenericOpenAIProvider(
        api_key=config.get("api_key"),
        base_url=config.get("base_url", "https://llm.mycompany.com/v1"),
    )
```

#### 3. 配置环境变量

```bash
OPENYAK_MY_LLM_API_KEY=your-api-key
```

#### 4. 重启后端

模型会自动出现在前端模型选择器中。

## 实战案例 4：添加前端页面

### 需求

添加一个使用统计页面。

### 实现步骤

#### 1. 创建 Hook

创建 `frontend/src/hooks/use-stats.ts`：

```typescript
import { useQuery } from '@tanstack/react-query';
import { api } from '@/lib/api';

export interface StatsOverview {
  total_sessions: number;
  total_messages: number;
  total_tokens: number;
  total_cost: number;
}

export function useStats() {
  return useQuery({
    queryKey: ['stats'],
    queryFn: () => api.get<StatsOverview>('/api/usage'),
    staleTime: 5 * 60 * 1000, // 5 分钟
  });
}
```

#### 2. 创建页面

创建 `frontend/src/app/(main)/stats/page.tsx`：

```typescript
'use client';

import { useStats } from '@/hooks/use-stats';

export default function StatsPage() {
  const { data, isLoading } = useStats();

  if (isLoading) {
    return <div className="p-6 text-[var(--text-secondary)]">加载中...</div>;
  }

  return (
    <div className="p-6">
      <h1 className="text-xl font-semibold text-[var(--text-primary)] mb-6">使用统计</h1>

      <div className="grid grid-cols-2 gap-4">
        <StatCard title="总会话数" value={data?.total_sessions} />
        <StatCard title="总消息数" value={data?.total_messages} />
        <StatCard title="总 Token 数" value={data?.total_tokens} />
        <StatCard title="总成本" value={`$${data?.total_cost?.toFixed(2)}`} />
      </div>
    </div>
  );
}

function StatCard({ title, value }: { title: string; value?: string | number }) {
  return (
    <div className="p-4 rounded-lg bg-[var(--surface-secondary)]">
      <div className="text-sm text-[var(--text-secondary)]">{title}</div>
      <div className="text-2xl font-bold text-[var(--text-primary)]">
        {typeof value === 'number' ? value.toLocaleString() : value ?? '-'}
      </div>
    </div>
  );
}
```

#### 3. 添加导航（可选）

在侧边栏导航组件中添加统计页链接。

## 实战案例 5：创建自定义插件

### 需求

创建一个 "项目管理" 插件，包含需求分析和迭代规划两个技能。

### 实现步骤

#### 1. 创建目录结构

```bash
mkdir -p data/plugins/project-mgmt/.claude-plugin
mkdir -p data/plugins/project-mgmt/skills/requirement-analysis/references
mkdir -p data/plugins/project-mgmt/skills/iteration-planning/references
```

#### 2. 创建 plugin.json

```json
{
  "name": "project-mgmt",
  "version": "1.0.0",
  "description": "项目管理技能包",
  "author": "Your Team",
  "skills": ["requirement-analysis", "iteration-planning"]
}
```

#### 3. 创建技能

`skills/requirement-analysis/SKILL.md`：

```markdown
---
name: requirement-analysis
description: 需求分析技能，帮助拆解和梳理项目需求
trigger: /req-analyze
---

# 需求分析技能

## Instructions

你是一位资深的需求分析师。当用户提出项目需求时：

1. **理解需求**：仔细阅读用户的需求描述
2. **拆解需求**：将大需求拆分为可执行的子需求
3. **识别依赖**：分析子需求之间的依赖关系
4. **评估优先级**：使用 MoSCoW 方法（Must/Should/Could/Won't）
5. **输出格式**：使用结构化 Markdown 输出

## Output Format

### 需求概述
[一句话描述]

### 子需求列表
| ID | 需求 | 优先级 | 依赖 | 预估 |
|----|------|--------|------|------|
| R1 | ... | Must | - | 2d |

### 风险点
- ...
```

#### 4. 安装和启用

```bash
# 通过 API 安装
POST /api/plugins/install
{"path": "data/plugins/project-mgmt"}

# 启用
POST /api/plugins/project-mgmt/enable
```

## 常见问题

### 工具注册后不生效？

确保在 `builtin/__init__.py` 的 `register_builtin_tools` 函数中调用了 `registry.register()`，然后重启后端。

### Agent 不出现在选择器中？

确保 agent 的 `mode` 设置为 `"primary"`，重启后端后前端会自动拉取。

### 插件技能不生效？

1. 确认 `plugin.json` 格式正确
2. 确认技能目录包含 `SKILL.md`
3. 通过 `POST /api/plugins/{id}/enable` 启用插件
4. 重启后端

### Provider 模型不显示？

1. 确认 API Key 已正确配置
2. 确认 `catalog.py` 中的 `env_key` 与环境变量名匹配
3. 重启后端
4. 检查后端日志是否有连接错误
