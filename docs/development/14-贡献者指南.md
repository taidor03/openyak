# 14. 贡献者指南

## 14.1 如何贡献

感谢您考虑为 OpenYak 项目做出贡献！本指南将帮助您了解如何参与项目开发。

## 14.2 贡献类型

### 14.2.1 代码贡献

- **新功能**：添加新的工具、Provider、Agent 等
- **bug 修复**：修复已报告的问题
- **性能优化**：提升系统性能
- **代码清理**：重构、优化代码结构

### 14.2.2 文档贡献

- **新增文档**：补充缺失的文档
- **文档改进**：改进现有文档的清晰度和准确性
- **翻译**：将文档翻译成其他语言

### 14.2.3 其他贡献

- **问题报告**：报告 bug 或提出改进建议
- **功能请求**：提出新功能想法
- **社区支持**：帮助其他用户解决问题

## 14.3 开发环境设置

### 14.3.1 Fork 项目

1. 访问 [https://github.com/openyak/openyak](https://github.com/openyak/openyak)
2. 点击右上角的 "Fork" 按钮
3. 克隆您的 fork：

```bash
git clone https://github.com/your-username/openyak.git
cd openyak
```

### 14.3.2 配置上游仓库

```bash
# 添加上游仓库
git remote add upstream https://github.com/openyak/openyak.git

# 验证远程仓库
git remote -v

# 定期同步上游代码
git fetch upstream
git checkout main
git merge upstream/main
```

### 14.3.3 安装开发依赖

```bash
# 后端依赖
cd backend
pip install -r requirements.txt
pip install -r requirements-dev.txt

# 前端依赖
cd ../frontend
npm install

# 安装开发工具
pip install black isort flake8 mypy pytest pytest-asyncio
```

### 14.3.4 配置本地环境

```bash
# 复制示例配置
cp backend/.env.example backend/.env

# 编辑配置
nano backend/.env

# 设置必要的变量
DATABASE_URL=postgresql+asyncpg://localhost/openyak_test
OPENROUTER_API_KEY=your_test_key
SECRET_KEY=test-secret-key
```

## 14.4 开发流程

### 14.4.1 创建分支

```bash
# 切换到 main 分支
git checkout main

# 拉取最新代码
git pull origin main

# 创建功能分支
git checkout -b feature/your-feature-name

# 或创建 bugfix 分支
git checkout -b fix/your-bugfix-name
```

### 14.4.2 编写代码

#### 代码规范

- 遵循 [代码规范文档](./09-代码规范.md)
- 使用类型注解
- 添加文档字符串
- 编写测试

#### 提交信息规范

```
<type>(<scope>): <subject>

<body>

<footer>
```

**类型（type）**：
- `feat`: 新功能
- `fix`: bug 修复
- `docs`: 文档更新
- `style`: 代码格式调整
- `refactor`: 代码重构
- `test`: 测试相关
- `chore`: 构建/工具相关

**示例**：
```
feat(tool): add weather query tool

- Implement WeatherTool class
- Add parameters for city and unit
- Add unit tests

Closes #123
```

### 14.4.3 运行测试

```bash
# 运行所有测试
pytest

# 运行特定测试
pytest tests/test_tool.py

# 运行测试并查看覆盖率
pytest --cov=app --cov-report=html

# 运行 lint 检查
flake8 app/
mypy app/

# 格式化代码
black app/
isort app/
```

### 14.4.4 提交更改

```bash
# 查看更改状态
git status

# 添加更改
git add .

# 提交更改
git commit -m "feat: add new feature"

# 推送分支
git push origin feature/your-feature-name
```

### 14.4.5 创建 Pull Request

1. 在 GitHub 上访问您的 fork
2. 点击 "Compare & pull request"
3. 填写 PR 模板：
   - 描述变更内容
   - 关联的 issue 编号
   - 测试说明
4. 等待代码审查

## 14.5 代码审查流程

### 14.5.1 审查标准

- [ ] 代码符合项目规范
- [ ] 有充分的测试覆盖
- [ ] 文档已更新
- [ ] 无明显的性能问题
- [ ] 安全性已考虑

### 14.5.2 审查反馈处理

1. **仔细阅读反馈**：理解审查者的意见
2. **逐条处理**：对每条反馈做出回应
3. **继续提交**：推送新的 commit 来修复问题
4. **保持沟通**：如有争议，礼貌讨论

### 14.5.3 审查检查清单

```markdown
## 代码质量
- [ ] 代码清晰易懂
- [ ] 命名规范
- [ ] 无重复代码
- [ ] 适当的错误处理

## 测试
- [ ] 单元测试覆盖
- [ ] 集成测试通过
- [ ] 边缘情况已处理

## 文档
- [ ] 代码注释充分
- [ ] API 文档已更新
- [ ] CHANGELOG 已更新

## 安全性
- [ ] 无安全漏洞
- [ ] 输入已验证
- [ ] 敏感信息已保护
```

## 14.7 测试指南

### 14.7.1 测试类型

#### 单元测试

```python
# tests/test_my_tool.py
import pytest
from app.tool.my_tool import MyTool

@pytest.mark.asyncio
async def test_my_tool():
    tool = MyTool()
    result = await tool.execute({"param": "value"})
    assert result.success
```

#### 集成测试

```python
# tests/test_integration.py
import pytest
from fastapi.testclient import TestClient
from app.main import app

@pytest.fixture
def client():
    return TestClient(app)

def test_chat_endpoint(client):
    response = client.post("/api/chat", json={"message": "test"})
    assert response.status_code == 200
```

### 14.7.2 测试覆盖率

```bash
# 运行测试并生成覆盖率报告
pytest --cov=app --cov-report=html --cov-report=term-missing

# 查看覆盖率
open htmlcov/index.html

# 要求覆盖率 > 80%
pytest --cov=app --cov-fail-under=80
```

### 14.7.3 测试最佳实践

- 测试命名清晰（test_*）
- 每个测试只测试一个功能
- 测试相互独立
- 使用 fixtures 共享测试数据
- 测试应快速执行

## 14.8 发布流程

### 14.8.1 版本管理

使用 [语义化版本](https://semver.org/)：

```
主版本。次版本.修订版本
MAJOR.MINOR.PATCH
```

- **MAJOR**: 不兼容的 API 修改
- **MINOR**: 向下兼容的功能新增
- **PATCH**: 向下兼容的问题修复

### 14.8.2 发布检查清单

```markdown
## 发布前检查

- [ ] 所有测试通过
- [ ] 文档已更新
- [ ] CHANGELOG 已更新
- [ ] 版本号已更新
- [ ] 无已知严重 bug

## 发布步骤

1. 切换到 main 分支
2. 拉取最新代码
3. 更新版本号
4. 提交版本标签
5. 推送标签
6. 创建 GitHub Release
```

### 14.8.3 CHANGELOG 格式

```markdown
# Changelog

## [1.1.0] - 2024-01-01

### Added
- 新增功能 1
- 新增功能 2

### Changed
- 变更 1
- 变更 2

### Fixed
- 修复 1
- 修复 2

### Security
- 安全更新

[1.1.0]: https://github.com/openyak/openyak/compare/v1.0.0...v1.1.0
```

## 14.10 常见问题

### Q1: 我不知道如何开始？

**A**: 从标记为 "good first issue" 的 issue 开始，这些 issue 适合新手。

### Q2: 我的 PR 长时间没有回应？

**A**: 礼貌地 ping 维护者，或者在 Discussions 中询问。

### Q3: 我的 PR 被拒绝了？

**A**: 不要灰心！仔细阅读反馈，理解原因，然后改进或尝试其他 issue。

### Q4: 我发现了严重 bug？

**A**: 立即报告 issue，并在描述中注明 "Security" 或 "Critical"。

### Q5: 我可以独立开发一个大功能？

**A**: 先在 Discussions 中提出想法，获得反馈后再开始开发。
