# OpenYak 构建与部署

## 概述

OpenYak 支持三种部署模式：
1. **开发模式**：前后端独立运行，热重载
2. **Web 部署**：前端静态导出 + 后端独立服务
3. **桌面应用**：Tauri v2 打包为原生安装包

## 构建系统总览

```
Monorepo 根目录 (package.json)
├── npm run dev:all        → 全栈开发
├── npm run dev:frontend   → 仅前端
├── npm run dev:backend    → 仅后端
├── npm run dev:desktop    → 桌面端开发
├── npm run build:frontend → 前端构建
├── npm run build:backend  → 后端构建 (PyInstaller)
├── npm run build:desktop  → 桌面端完整构建
└── npm run build:installer → 安装包构建
```

## 前端构建

### 开发模式

```bash
cd frontend
npm run dev
# → http://localhost:3000 (Turbopack HMR)
```

### 生产构建

```bash
cd frontend
npm run build
# → 输出到 frontend/out/ (静态导出)
```

Next.js 配置为 `output: 'export'`，生成纯静态文件到 `out/` 目录。这是因为桌面端需要嵌入静态文件。

**关键配置** (`next.config.ts`)：
```typescript
const nextConfig = {
  output: 'export',
  images: { unoptimized: true }, // 静态导出不支持图片优化
  // ...
};
```

### 构建产物

```
frontend/out/
├── index.html
├── _next/
│   ├── static/        # JS/CSS/字体
│   └── ...
├── c/
│   └── [id]/          # 对话页
├── settings/          # 设置页
└── ...
```

### 环境变量

```bash
# 开发环境
NEXT_PUBLIC_API_URL=http://localhost:8000

# 桌面构建
NEXT_PUBLIC_DESKTOP_BUILD=true
NEXT_PUBLIC_API_URL=http://localhost:8000

# 生产 Web
NEXT_PUBLIC_API_URL=https://api.your-domain.com
```

## 后端构建

### 开发模式

```bash
cd backend
uvicorn app.main:create_app --factory --reload --reload-dir app --host 0.0.0.0 --port 8000
```

或从根目录：
```bash
npm run dev:backend
```

### 生产构建 (PyInstaller)

```bash
cd backend
pyinstaller openyak.spec
```

**openyak.spec** 关键配置：
- 单文件模式 (`--onefile`)
- 包含 `resources/nodejs/` 目录
- 包含 `data/plugins/` 目录
- 入口点: `run.py`

### 构建产物

```
backend/dist/openyak-backend/
├── openyak-backend      # 可执行文件
└── _internal/           # 依赖库
    ├── Python 运行时
    ├── site-packages
    └── 资源文件
```

### 运行生产后端

```bash
./dist/openyak-backend/openyak-backend --port 8000 --data-dir /path/to/data
```

## 桌面端构建

### 完整构建流程

桌面端构建是最复杂的，涉及前端静态导出 → 后端打包 → Tauri 集成。

```bash
# 一键构建
npm run build:desktop
```

或手动分步：

```bash
# 1. 构建前端（静态导出）
cd frontend
NEXT_PUBLIC_DESKTOP_BUILD=true npm run build
cd ..

# 2. 构建后端（PyInstaller）
cd backend
pyinstaller openyak.spec
cd ..

# 3. 同步桌面元数据
node scripts/sync-desktop-meta.mjs

# 4. Tauri 构建
cd desktop-tauri
npm run tauri build
```

### build-desktop.sh 脚本详解

`scripts/build-desktop.sh` 是完整的桌面端构建脚本（526 行），执行以下步骤：

```bash
#!/bin/bash
# 1. 环境检查（Node/Python/Rust）
# 2. 清理旧构建产物
# 3. 安装前端依赖
# 4. 构建前端（NEXT_PUBLIC_DESKTOP_BUILD=true）
# 5. 安装后端依赖
# 6. 下载 Node.js 运行时（用于后端 MCP 功能）
# 7. 更新技能目录
# 8. 构建后端（PyInstaller）
# 9. 安装 Tauri 依赖
# 10. 同步桌面元数据
# 11. Tauri 构建（生成安装包）
# 12. 校验和生成
```

### 平台特定构建

Tauri 支持跨平台构建，但需要在对应平台上编译：

```bash
# macOS (Apple Silicon)
cd desktop-tauri
npm run tauri build -- --target aarch64-apple-darwin

# macOS (Intel)
npm run tauri build -- --target x86_64-apple-darwin

# Windows
npm run tauri build -- --target x86_64-pc-windows-msvc

# Linux
npm run tauri build -- --target x86_64-unknown-linux-gnu
```

**构建配置文件**：
- `desktop-tauri/build.macos-aarch64.json` — macOS ARM64
- `desktop-tauri/build.macos-x64.json` — macOS Intel
- `desktop-tauri/build.windows.json` — Windows
- `desktop-tauri/build.linux-x64.json` — Linux

### 构建产物

```
desktop-tauri/src-tauri/target/release/bundle/
├── dmg/          # macOS DMG 安装包
│   └── OpenYak_1.1.9_aarch64.dmg
├── msi/          # Windows MSI 安装包
│   └── OpenYak_1.1.9_x64.msi
└── deb/          # Linux DEB 包
    └── openyak_1.1.9_amd64.deb
```

### 校验和生成

```bash
node scripts/generate-checksums.mjs
# → 生成 SHA256 校验和文件
```

## 版本管理

### 版本号更新

```bash
node scripts/bump-version.mjs 1.2.0
```

此脚本会更新以下文件中的版本号：
- `package.json` (根目录)
- `frontend/package.json`
- `desktop-tauri/package.json`
- `desktop-tauri/src-tauri/tauri.conf.json`
- `backend/pyproject.toml`

### 版本号格式

遵循 Semantic Versioning：`MAJOR.MINOR.PATCH`

- **MAJOR**：不兼容的 API 变更
- **MINOR**：向后兼容的新功能
- **PATCH**：向后兼容的 bug 修复

## 部署模式

### 模式 1：本地桌面应用

最常用的部署方式。用户下载安装包，本地运行。

```
┌──────────────────────────────┐
│        Desktop App           │
│  ┌─────────┐ ┌────────────┐ │
│  │ Tauri   │ │ Frontend   │ │
│  │ Shell   │→│ (embedded) │ │
│  └─────────┘ └────────────┘ │
│  ┌─────────┐ ┌────────────┐ │
│  │ Backend │ │ SQLite DB  │ │
│  │ (embed) │→│ (local)    │ │
│  └─────────┘ └────────────┘ │
└──────────────────────────────┘
```

**安装步骤**：
1. 下载对应平台的安装包
2. 安装到本地
3. 启动应用
4. 配置 API Key（至少一个）
5. 开始使用

### 模式 2：Web 服务部署

将 OpenYak 部署为 Web 服务，支持多用户远程访问。

```
┌──────────────────────────────┐
│         Web Browser          │
│  ┌─────────────────────────┐ │
│  │   Frontend (CDN/Nginx)  │ │
│  └────────────┬────────────┘ │
└───────────────┼──────────────┘
                │ HTTPS
┌───────────────▼──────────────┐
│        Backend Server        │
│  ┌─────────┐ ┌────────────┐ │
│  │ FastAPI │ │ SQLite DB  │ │
│  │ Server  │→│ (local)    │ │
│  └─────────┘ └────────────┘ │
└──────────────────────────────┘
```

**部署步骤**：

```bash
# 1. 构建前端
cd frontend
NEXT_PUBLIC_API_URL=https://api.your-domain.com npm run build

# 2. 部署前端到 Nginx/CDN
cp -r out/* /var/www/openyak/

# 3. 配置 Nginx
# /etc/nginx/sites-available/openyak
server {
    listen 443 ssl;
    server_name your-domain.com;
    root /var/www/openyak;
    
    location / {
        try_files $uri $uri.html $uri/ =404;
    }
    
    location /api/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
    
    location /api/chat/stream/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Connection '';
        proxy_http_version 1.1;
        chunked_transfer_encoding off;
        proxy_buffering off;
        proxy_cache off;
    }
}

# 4. 运行后端
cd backend
OPENYAK_HOST=127.0.0.1 \
OPENYAK_PORT=8000 \
OPENYAK_DEBUG=false \
OPENYAK_REMOTE_ACCESS_ENABLED=true \
uvicorn app.main:create_app --factory --host 127.0.0.1 --port 8000
```

### 模式 3：远程隧道访问

通过 Cloudflare Tunnel 从外部访问本地桌面端。

```
Mobile Browser
  ↓ HTTPS
Cloudflare Tunnel (自动配置)
  ↓
Desktop Backend (localhost:8000)
  ↓
SQLite (本地)
```

**配置**：

```bash
# .env
OPENYAK_REMOTE_ACCESS_ENABLED=true
OPENYAK_REMOTE_TUNNEL_MODE=auto
```

或手动指定隧道 URL：
```bash
OPENYAK_REMOTE_ACCESS_ENABLED=true
OPENYAK_REMOTE_TUNNEL_MODE=manual
OPENYAK_REMOTE_TUNNEL_URL=https://your-tunnel.trycloudflare.com
```

## Docker 部署（社区方案）

虽然 OpenYak 官方不提供 Docker 镜像，但可以自行构建：

```dockerfile
FROM python:3.12-slim

WORKDIR /app

# 安装后端
COPY backend/ .
RUN pip install -e ".[dev]"

# 复制前端构建产物
COPY frontend/out/ /app/static/

# 配置
ENV OPENYAK_HOST=0.0.0.0
ENV OPENYAK_PORT=8000
ENV OPENYAK_DATABASE_URL=sqlite+aiosqlite:///./data/openyak.db

EXPOSE 8000

CMD ["uvicorn", "app.main:create_app", "--factory", "--host", "0.0.0.0", "--port", "8000"]
```

## 构建故障排查

### 前端构建失败

```bash
# 清理缓存
rm -rf frontend/.next frontend/out
cd frontend && npm run build
```

### PyInstaller 构建失败

```bash
# 清理旧构建
rm -rf backend/build backend/dist
cd backend && pyinstaller openyak.spec
```

常见问题：
- **缺少模块**：在 `openyak.spec` 的 `hiddenimports` 中添加
- **文件未打包**：在 `openyak.spec` 的 `datas` 中添加

### Tauri 构建失败

```bash
# 确保 Rust 工具链最新
rustup update stable

# 清理 Rust 构建缓存
rm -rf desktop-tauri/src-tauri/target
cd desktop-tauri && npm run tauri build
```

### Node.js 运行时下载失败

后端 MCP 功能需要 Node.js 运行时。如果下载失败：

```bash
cd backend
python scripts/download_node.py
```

### 校验和不匹配

```bash
node scripts/verify-bundle.mjs
```
