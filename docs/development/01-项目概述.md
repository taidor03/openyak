# OpenYak 项目概述

## 项目简介

OpenYak 是一个**本地优先的 AI 工作台**，专为真实办公场景设计。它不是简单的聊天框，而是一个可以处理文件、执行工具、管理 artifact 的完整 agent 运行时。用户可以在本地桌面端运行，通过 BYOK（Bring Your Own Key）或本地模型（Ollama）获得完全隐私控制的 AI 体验。

### 核心特性

- **本地优先**：文件、对话、记忆和生成结果都存储在本机，零数据上传
- **Agent 系统**：内置 7 个 agent（build/plan/explore/general/compaction/title/summary），支持多步工具调用和长上下文任务
- **工具生态**：20+ 内置工具（read/write/edit/bash/glob/grep/search/code_execute 等），支持 MCP 协议扩展
- **多模型支持**：21 个 BYOK 提供商 + Ollama 本地 + ChatGPT 订阅接入
- **上下文管理**：三层压缩策略（microcompact → context collapse → LLM compaction），自动管理长对话
- **文件处理**：支持 DOCX、XLSX、PPTX、PDF、CSV 等多种格式的读写
- **Artifact 系统**：生成和展示 Markdown、代码、Mermaid 流程图、表格等工件
- **记忆系统**：自动提取对话中的关键事实，跨会话持久化
- **远程访问**：通过 Cloudflare Tunnel 从手机/浏览器连接桌面端
- **自动化**：Cron 定时任务，自动执行预设 prompt
- **插件系统**：Cowork Plugin 架构，可安装领域专用技能包

## 核心概念

### Agent（智能体）

Agent 是 OpenYak 的核心执行单元。每个 agent 有独立的：

- **System Prompt**：定义 agent 的行为和目标（存储在 `agent/prompts/{name}.txt`）
- **工具权限**：通过 `Ruleset` 控制可访问哪些工具（allow/deny/ask 三种动作）
- **工具列表**：`tools=[]` 表示所有工具，也可限定为特定子集
- **模式**：`primary`（用户可选）/ `subagent`（由 task 工具启动）/ `hidden`（系统内部）

### Tool（工具）

工具是 agent 执行具体操作的接口。每个工具继承 `ToolDefinition` 基类：

- 实现 `execute(args, ctx) -> ToolResult` 方法
- 定义 JSON Schema 参数（`parameters_schema()`）
- 支持并发安全标记（`is_concurrency_safe`）
- 可附加元数据和文件附件

### Provider（提供商）

Provider 是 LLM 服务的抽象层，所有提供商实现 `BaseProvider` 接口：

- `stream_chat(messages, model, **kwargs)` — 流式生成
- `all_models()` — 返回可用模型列表
- 工具调用适配（原生函数调用 vs prompt-based 回退）

### Session（会话）

Session 是对话的容器，消息以 `Message + Part` 两级结构存储：

- **Message**：一条用户或助手消息，包含 `data` JSON 字段
- **Part**：消息的组成部分（text/reasoning/tool/step_start/step_finish/compaction/file 等）
- **分页**：Infinite Query 反向分页，`offset=-1` 获取最新页

### Artifact（工件）

Artifact 是可复用的内容块，支持 Markdown、代码、Mermaid 流程图、CSV/XLSX 表格、PDF/DOCX/PPTX 文档预览。

### Memory（记忆）

自动提取对话中的关键事实，持久化到 `Fact` 表，在后续会话的 System Prompt 中注入。

## 数据流总览

### 用户输入到 AI 响应

```
用户输入（ChatForm）
  ↓ POST /api/chat/prompt
SessionPrompt（编排器）
  ↓ 创建 UserMessage + 构建 System Prompt
Agent Loop（SessionPrompt.run()）:
  │
  ├─ SessionProcessor（单步执行器）
  │   ├─ stream_llm() → 流式调用 LLM
  │   ├─ 解析 tool_calls
  │   ├─ 权限检查（evaluate()）
  │   ├─ doom loop 检测
  │   ├─ 执行工具（ToolExecutor）
  │   └─ 返回 "continue" | "stop" | "compact"
  │
  ├─ SSE 事件流 → 前端实时渲染
  │
  └─ 循环直到无工具调用或用户中止
  ↓
自动生成标题 / 触发上下文压缩
  ↓
发布 done 事件 → 前端同步数据库状态
```

### SSE 流式传输

```
POST /api/chat/prompt → {stream_id, session_id}
  ↓
EventSource → /api/chat/stream/{stream_id}
  ↓
SSE 事件流:
  ├─ step_start       → 步骤开始
  ├─ text_delta       → 文本增量
  ├─ reasoning_delta  → 推理增量
  ├─ tool_start       → 工具开始执行
  ├─ tool_result      → 工具执行结果
  ├─ tool_error       → 工具执行错误
  ├─ step_finish      → 步骤完成（含 token/cost）
  ├─ permission_request → 权限请求
  ├─ question         → 问题提示
  ├─ compaction_*     → 上下文压缩进度
  ├─ agent_error      → Agent 异常
  ├─ done             → 生成完成
  └─ error            → 错误
```

## 开发环境

### 前置要求

| 工具 | 版本 | 用途 |
|------|------|------|
| Node.js | 18+ | 前端构建 & 开发脚本 |
| Python | 3.12+ | 后端运行 |
| Rust | 1.70+ | 桌面端开发（仅桌面端需要） |
| Git | 2.x | 版本控制 |

### 快速启动

```bash
# 1. 克隆仓库
git clone https://github.com/openyak/openyak.git
cd openyak

# 2. 安装依赖
npm install                          # 前端 + 脚本依赖
cd backend && pip install -e ".[dev]" && cd ..  # 后端依赖

# 3. 配置环境
cp backend/.env.example backend/.env  # 后端配置
# 编辑 .env 添加 API Key（至少一个）

# 4. 启动全栈开发
npm run dev:all
# 后端：http://localhost:8000
# 前端：http://localhost:3000
```

### 其他开发模式

```bash
npm run dev:frontend   # 仅前端（需要后端已运行）
npm run dev:backend    # 仅后端
npm run dev:desktop    # 桌面端开发模式
```
