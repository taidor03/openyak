# OpenYak 前端开发指南

## 开发环境设置

### 前置要求

| 工具 | 版本 | 说明 |
|------|------|------|
| Node.js | 18+ | 前端运行和构建 |
| npm | 最新 | 包管理 |

### 安装依赖

```bash
cd frontend
npm install
```

### 配置环境变量

创建 `.env.local` 文件：

```bash
NEXT_PUBLIC_API_URL=http://localhost:8000
```

### 启动开发服务器

```bash
cd frontend
npm run dev
# → http://localhost:3000 (Turbopack HMR)

# 或从项目根目录
npm run dev:frontend
```

## 核心概念

### 组件架构分层

```
1. UI Primitives (components/ui/)
   └── shadcn/ui 基础组件，零业务逻辑
       ├── Button, Dialog, Dropdown
       ├── Input, Select, Switch
       └── ...

2. Feature Components (components/<feature>/)
   └── 业务组件，可导入 UI primitives
       ├── chat/ → 聊天输入、提交
       ├── messages/ → 消息列表、分组
       ├── parts/ → 消息部件渲染
       ├── settings/ → 设置页面
       └── ...

3. Page Components (app/)
   └── 页面组件，组合 feature components
       ├── c/[id]/page.tsx → 对话页
       └── settings/page.tsx → 设置页
```

**规则**：
- `feature/` 可以导入 `ui/`，反之不行
- 使用 Tailwind utilities + CSS 变量，不使用 inline styles
- 使用 `cn()` 函数组合类名
- 使用 CVA (class-variance-authority) 处理变体

### 状态管理策略

```
┌─────────────────────────────────────────────────────────┐
│                    状态管理决策树                        │
│                                                         │
│  数据是否来自服务器？                                    │
│  ├── 是 → TanStack Query                                │
│  │   ├── 会话列表、消息、模型...                         │
│  │   ├── 自动缓存、后台刷新、SSE 驱动 invalidate        │
│  │   └── 使用 hooks/use-*.ts 封装                       │
│  │                                                      │
│  └── 否 → Zustand Store                                 │
│      ├── 流式状态、UI 开关、用户偏好...                  │
│      ├── 全局可访问、无 prop drilling                    │
│      └── 使用 stores/*-store.ts                         │
└─────────────────────────────────────────────────────────┘
```

### SSE 流式传输

SSE 是前端与后端实时通信的核心机制：

```typescript
// hooks/use-sse.ts 简化逻辑
export function useSse(streamId: string | null) {
  const chatStore = useChatStore();

  useEffect(() => {
    if (!streamId) return;

    const eventSource = new EventSource(
      `/api/chat/stream/${streamId}`
    );

    eventSource.addEventListener('text_delta', (e) => {
      chatStore.appendTextDelta(JSON.parse(e.data));
    });

    eventSource.addEventListener('tool_start', (e) => {
      chatStore.addToolStart(JSON.parse(e.data));
    });

    eventSource.addEventListener('done', () => {
      chatStore.setStreaming(false);
      // 刷新数据库状态
      queryClient.invalidateQueries({ queryKey: ['messages'] });
      eventSource.close();
    });

    return () => eventSource.close();
  }, [streamId]);
}
```

### 消息渲染系统

消息采用两级结构：Message → Parts

```
Message
├── role: "user" | "assistant"
├── data: { collapsed: true }  ← 上下文折叠标记
└── parts: Part[]
    ├── TextPart        → parts/text-part.tsx
    ├── ReasoningPart   → parts/reasoning-part.tsx (可折叠)
    ├── ToolPart        → parts/tool-part.tsx (可折叠)
    ├── StepStartPart   → parts/step-part.tsx
    ├── StepFinishPart  → parts/step-part.tsx
    ├── CompactionPart  → messages/message-group.tsx (折叠提示条)
    └── FilePart        → parts/file-part.tsx
```

### 无限滚动分页

消息列表使用 TanStack Query 的 `useInfiniteQuery`：

```typescript
// hooks/use-messages.ts
export function useMessages(sessionId: string) {
  return useInfiniteQuery({
    queryKey: ['messages', sessionId],
    queryFn: ({ pageParam }) =>
      api.get(`/api/messages/${sessionId}?offset=${pageParam}`),
    initialPageParam: -1,  // -1 = 最新页
    getNextPageParam: (lastPage) => lastPage.next_offset,
    maxPages: 5,
  });
}
```

### 创建 TanStack Query Hook

```typescript
// hooks/use-my-feature.ts
import { useQuery, useMutation, useQueryClient } from '@tanstack/react-query';
import { api } from '@/lib/api';

export function useMyFeature(id: string) {
  return useQuery({
    queryKey: ['myFeature', id],
    queryFn: () => api.get(`/api/myfeature/${id}`),
    staleTime: 30_000, // 30s
  });
}

export function useUpdateMyFeature() {
  const queryClient = useQueryClient();

  return useMutation({
    mutationFn: (data: { id: string; name: string }) =>
      api.patch(`/api/myfeature/${data.id}`, data),
    onSuccess: () => {
      queryClient.invalidateQueries({ queryKey: ['myFeature'] });
    },
  });
}
```

### CSS 变量主题

使用 CSS 变量实现主题切换，变量在 `globals.css` 中定义：

```css
:root {
  /* 文字 */
  --text-primary: #1a1a1a;
  --text-secondary: #666666;
  --text-tertiary: #999999;

  /* 表面 */
  --surface-primary: #ffffff;
  --surface-secondary: #f5f5f5;
  --surface-tertiary: #eeeeee;

  /* 品牌 */
  --brand-primary: #6366f1;
  --brand-secondary: #818cf8;
}

.dark {
  --text-primary: #e5e5e5;
  --text-secondary: #a3a3a3;
  --surface-primary: #1a1a1a;
  --surface-secondary: #262626;
  /* ... */
}
```

**使用规则**：

```typescript
// ✅ 正确：使用 CSS 变量
<div className="bg-[var(--surface-primary)] text-[var(--text-primary)]">

// ❌ 错误：硬编码颜色
<div className="bg-white text-gray-900">

// ✅ 正确：Tailwind utilities
<div className="p-4 rounded-lg bg-[var(--surface-secondary)]">

// ❌ 错误：inline styles
<div style={{ padding: '1rem', borderRadius: '0.5rem' }}>
```

## 性能优化

### 关键策略

1. **虚拟滚动**：TanStack Virtual 渲染可见区域消息
2. **无限滚动**：`useInfiniteQuery` 分页加载历史消息
3. **SSE 驱动刷新**：仅在 `done` 事件后 invalidate，避免轮询
4. **代码分割**：Next.js 自动按路由分割
5. **React.memo**：消息部件使用 memo 避免重渲染
6. **防抖节流**：输入、滚动、resize 事件

## 最佳实践

### 组件设计

- 单一职责，保持小而专注
- 使用 props 传递数据，回调处理事件
- 优先组合而非继承

### 状态管理

- 服务端数据用 TanStack Query（自动缓存和刷新）
- 客户端 UI 状态用 Zustand（全局可访问）
- 避免过度使用 React Context

### 样式

- 使用 Tailwind utilities + CSS 变量
- 不使用 CSS Modules
- 不使用 inline styles
- 使用 `cn()` 组合条件类名
