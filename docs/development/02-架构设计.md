# OpenYak 架构设计

## 整体架构

OpenYak 采用三层架构：桌面外壳、前端 UI、后端引擎。三层之间通过 HTTP/SSE 通信，各自可独立部署。

```
┌────────────────────────────────────────────────────────────────┐
│                    Desktop Shell (Tauri v2)                    │
│  窗口管理 │ 系统托盘 │ 原生对话框 │ 文件系统 │ 剪贴板 │ 通知    │
└───────────────────────┬────────────────────────────────────────┘
                        │ HTTP/WebView
┌───────────────────────▼────────────────────────────────────────┐
│                  Frontend (Next.js 15 SSG)                     │
│  聊天 UI │ 设置 │ Artifact 面板 │ SSE 流 │ 状态管理 │ 文件上传   │
└───────────────────────┬────────────────────────────────────────┘
                        │ HTTP / SSE
┌───────────────────────▼────────────────────────────────────────┐
│                   Backend (FastAPI)                             │
│  Agent 引擎 │ 工具执行 │ LLM 流式 │ 数据持久化 │ MCP │ 记忆     │
└────────────────────────────────────────────────────────────────┘
```

## 核心进程模型

```
Desktop App
├── Tauri Main Process (Rust)     ← 窗口管理、系统调用
├── WebView Process               ← 渲染 Next.js 静态导出
│   └── Frontend (localhost:3000 或 embedded)
└── Backend Process (Python)      ← FastAPI 服务
    └── SQLite (本地文件 data/openyak.db)
```

### 远程访问模式

```
Mobile Browser
  ↓ HTTPS
Cloudflare Tunnel
  ↓
Desktop Backend (localhost:8000)
  ↓
SQLite (本地)
```

## 核心子系统

```

  │   ├── build (primary)   — 全功能助手
  │   ├── plan (primary)    — 只读分析模式
  │   ├── explore (subagent) — 快速搜索
  │   ├── general (subagent) — 通用型
  │   ├── compaction (hidden) — 上下文摘要
  │   ├── title (hidden)     — 自动标题
  │   └── summary (hidden)   — 摘要统计
  ├── 权限引擎 (4 层: global → agent → user → session)
  └── 工具注册表 (通过 agent.tools 过滤)
```

### 工具系统

```
ToolRegistry
  ├── ToolDefinition (抽象基类)
  │   ├── id, description, is_concurrency_safe
```

### 会话处理系统（核心循环）

```
SessionPrompt (编排器)              SessionProcessor (单步执行器)
┌─────────────────────────┐        ┌─────────────────────────┐
│ _setup()                │        │ __call__()              │
│   解析 agent/model      │        │   stream_llm()          │
│   构建 system_prompt    │  →→→   │   解析 tool_calls       │
│   加载 workspace memory │        │   权限检查              │
│                         │        │   doom loop 检测        │
│ run() — Agent Loop      │        │   执行工具              │
│   while True:           │        │   保存 Part             │
│     processor = new     │        │   返回 Continue/Stop/   │
│     result = processor()│        │         Compact         │
│     if Stop: break      │        └─────────────────────────┘
│     if Compact: compact │
│   保存 assistant msg    │
│   触发 title/summary    │
└─────────────────────────┘
```

### 上下文压缩系统（三层策略）

```
Layer 1: Microcompact（微压缩）
  ├── 触发: 每次请求前
  ├── 策略: 移除旧 tool 输出的冗余部分
  └── 位置: session/microcompact.py

Layer 2: Context Collapse（折叠）
  ├── 触发: 消息数超过阈值
  ├── 策略: 标记旧消息为 collapsed，插入 boundary marker
  └── 位置: session/prompt.py (_persist_context_collapse)

Layer 3: LLM Compaction（LLM 摘要压缩）
  ├── 触发: token 超过上下文窗口 85%
  ├── 策略: 使用 compaction agent 生成摘要，替换旧消息
  └── 位置: session/compaction.py
```

## 数据流详解

### 完整请求生命周期

```
1. 用户输入 "帮我分析这个项目"
   ↓
2. Frontend: ChatForm.onSubmit()
   → POST /api/chat/prompt {text, session_id, model, agent, files}
   ↓
3. Backend: SessionPrompt.__init__()
   → 创建 GenerationJob
   → 返回 {stream_id, session_id}
   ↓
4. Frontend: EventSource(/api/chat/stream/{stream_id})
   ↓
5. Backend: SessionPrompt._setup()
   → 解析 agent/model/provider
   → 构建 system_prompt（agent prompt + tools + memory + workspace）
   → 创建 UserMessage + UserPart
   ↓
6. Backend: SessionPrompt.run() — Agent Loop
   │
   ├─ SessionProcessor.__call__()
   │   → stream_llm() → 流式生成
   │   → SSE: step_start
   │   → SSE: text_delta × N
   │   → SSE: reasoning_delta × N（如果模型支持）
   │   → 解析 tool_calls
   │   │
   │   ├─ 如果有工具调用:
   │   │   → SSE: tool_start
   │   │   → 权限检查 evaluate()
   │   │   → 如果需要确认: SSE: permission_request → 等待用户响应
   │   │   → doom loop 检测
   │   │   → ToolExecutor.execute() → ToolResult
   │   │   → SSE: tool_result / tool_error
   │   │   → 保存 ToolPart
   │   │   → SSE: step_finish
   │   │   → 返回 Continue → 继续循环
   │   │
   │   └─ 如果无工具调用:
   │       → 保存 TextPart + ReasoningPart
   │       → SSE: step_finish
   │       → 返回 Stop → 结束循环
   │
   ├─ 如果需要压缩: compact → SSE: compaction_*
   │
   └─ 循环结束
   ↓
7. Backend: 后处理
   → 保存 AssistantMessage
   → 异步生成标题（title agent）
   → 异步计算摘要（summary agent）
   → 异步提取记忆（memory updater）
   → SSE: done
   ↓
8. Frontend: 收到 done
   → invalidateQueries(['messages']) — 刷新数据库状态
   → invalidateQueries(['sessions']) — 更新标题
```

### 文件上传流程

```
1. 用户选择文件（拖拽或点击）
   ↓
2. Frontend: POST /api/files/upload (multipart/form-data)
   ↓
3. Backend: files.py
   → 保存文件到工作区
   → 解析内容（DOCX/XLSX/PPTX/PDF/CSV 等）
   → 返回 FilePart 数据
   ↓
4. Frontend: 附加到 ChatForm
   → 在 prompt 请求中包含 files 数组
   ↓
5. Backend: SessionPrompt
   → 创建 FilePart 并附加到 UserMessage
   → 文件内容作为上下文发送给 LLM
```

## 安全架构

### 认证

- **本地**：Session Token（自动生成，存储在 `data/session_token.txt`）
- **远程**：JWT Token（通过 Cloudflare Tunnel）
- **OAuth**：ChatGPT 订阅、MCP 服务器、Google 认证

### 授权

4 层权限引擎，按优先级评估：

```
1. 全局权限 (GLOBAL_RULES)
   ↓
2. Agent 权限 (agent.permissions.rules)
   ↓
3. 用户权限 (user_permissions, 来自设置)
   ↓
4. 会话权限 (session_permissions, 来自对话设置)
```

动作类型：
- `allow` — 直接执行
- `deny` — 直接拒绝
- `ask` — 发送 permission_request SSE 事件，等待用户确认

### 中间件链

```python
app = FastAPI()
app.add_middleware(CsrfProtectionMiddleware)      # CSRF 保护
app.add_middleware(PrivateNetworkAccessMiddleware) # 私有网络访问控制
app.add_middleware(SessionTokenMiddleware)         # Session Token 认证
```

## 性能优化

### 前端

- **虚拟滚动**：TanStack Virtual（消息列表按需渲染）
- **无限滚动**：TanStack Query infiniteQuery（消息分页加载）
- **代码分割**：Next.js 自动 code splitting
- **SSE 驱动刷新**：只在 done 事件后 invalidate，避免轮询
- **防抖/节流**：输入、滚动、resize 事件

### 后端

- **全异步 I/O**：async/await 贯穿整个调用链
- **流式传输**：SSE 逐 token 推送，不等完整响应
- **上下文压缩**：三层策略减少 token 消耗
- **连接池**：httpx.AsyncClient 复用连接
- **FTS5 索引**：SQLite 全文搜索加速

### 数据库

- **WAL 模式**：读写并发
- **FTS5 全文搜索**：搜索不走 LIKE
- **批量写入**：消息 Part 批量 INSERT
- **JSON data 字段**：灵活扩展，减少 schema 变更
