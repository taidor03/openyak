# OpenYak 配置和环境文档

## 概述

OpenYak 通过环境变量进行配置，使用 Pydantic Settings 管理所有配置项。配置文件位于 `backend/app/config.py`（约 950 行），环境变量统一使用 `OPENYAK_` 前缀。

## 配置管理

### Pydantic Settings

```python
# backend/app/config.py
from pydantic_settings import BaseSettings

class Settings(BaseSettings):
    # 所有配置项定义在这里
    database_url: str = "sqlite+aiosqlite:///./data/openyak.db"
    host: str = "0.0.0.0"
    port: int = 8000
    # ...

    class Config:
        env_prefix = "OPENYAK_"  # 环境变量前缀
```

### 环境变量命名规则

所有环境变量以 `OPENYAK_` 为前缀，字段名自动大写：

```python
# Python 字段 → 环境变量
database_url  → OPENYAK_DATABASE_URL
host          → OPENYAK_HOST
port          → OPENYAK_PORT
```

## 数据库配置

### SQLite（默认，推荐本地使用）

```bash
OPENYAK_DATABASE_URL=sqlite+aiosqlite:///./data/openyak.db
```

特性：WAL 模式、FTS5 全文搜索、零配置。

### PostgreSQL（可选，适合多用户 Web 部署）

```bash
OPENYAK_DATABASE_URL=postgresql+asyncpg://user:password@localhost/openyak
```

### MySQL（可选）

```bash
OPENYAK_DATABASE_URL=mysql+aiomysql://user:password@localhost/openyak
```

## .env 文件

### 后端

```bash
# backend/.env
# 复制自 .env.example 并按需修改

OPENYAK_DATABASE_URL=sqlite+aiosqlite:///./data/openyak.db

# OPENYAK_ANTHROPIC_API_KEY=sk-ant-...

# Ollama
OPENYAK_OLLAMA_AUTO_START=true

# 功能
OPENYAK_COMPACTION_AUTO=true
OPENYAK_FTS_ENABLED=true
```

### 前端

```bash
# frontend/.env.local
NEXT_PUBLIC_API_URL=http://localhost:8000
```

## 安全配置

### Session Token

本地自动生成，无需配置。开发环境可手动指定：

```bash
OPENYAK_ALLOW_DEV_SESSION_TOKEN=true
OPENYAK_DEV_SESSION_TOKEN=my-dev-token
```

### CORS

添加额外允许的源：

```bash
OPENYAK_EXTRA_ALLOWED_ORIGINS=https://example.com,https://app.example.com
```

### 中间件链

```python
# main.py 中的中间件注册顺序
app.add_middleware(CsrfProtectionMiddleware)      # CSRF 保护
app.add_middleware(PrivateNetworkAccessMiddleware) # 私有网络访问控制
app.add_middleware(SessionTokenMiddleware)         # Session Token 认证
```

## 故障排查

### 数据库锁定

```bash
sqlite3 data/openyak.db "PRAGMA journal_mode;"
# 应返回 "wal"
```

### 端口冲突

```bash
OPENYAK_PORT=8001
```

### API 连接失败

1. 后端是否运行：`curl http://localhost:8000/health`
2. 前端 `NEXT_PUBLIC_API_URL` 是否正确
3. CORS 配置是否正确

### Ollama 连接失败

1. Ollama 是否安装：`ollama --version`
2. `OPENYAK_OLLAMA_BASE_URL` 是否正确
3. Ollama 服务是否运行：`curl http://localhost:11434/api/tags`
