# OpenYak 提供商集成文档

## 概述

提供商（Provider）是 LLM 服务的抽象层。OpenYak 支持 21 个 BYOK 提供商、Ollama 本地模型和 ChatGPT 订阅接入。所有提供商实现 `BaseProvider` 接口。

## 提供商架构

### BaseProvider 基类

**位置**：`backend/app/provider/base.py`

```python
class BaseProvider(ABC):
    @abstractmethod
    async def stream_chat(
        self,
        messages: list[ChatMessage],
        model: str,
        **kwargs
    ) -> AsyncGenerator[StreamChunk, None]:
        """流式聊天"""
        pass

    @abstractmethod
    async def all_models(self) -> list[ModelInfo]:
        """获取可用模型列表"""
        pass

    @abstractmethod
    def supports_tool_calling(self, model: str) -> bool:
        """是否支持原生工具调用"""
        pass
```

### 核心数据结构

```python
@dataclass
class ChatMessage:
    role: str  # "user", "assistant", "system"
    content: str | list[dict]  # 文本或多模态内容

@dataclass
class StreamChunk:
    delta: str | None = None           # 文本增量
    reasoning: str | None = None       # 推理增量
    tool_calls: list[dict] | None = None  # 工具调用
    finish_reason: str | None = None   # 完成原因

@dataclass
class ModelInfo:
    id: str
    name: str
    provider: str
    context_window: int
    capabilities: dict  # {"vision": bool, "reasoning": bool}
```

### ProviderRegistry

**位置**：`backend/app/provider/registry.py`

```python
class ProviderRegistry:
    def register(self, provider_id: str, provider: BaseProvider) -> None
    def get(self, provider_id: str) -> BaseProvider | None
    def list_providers(self) -> list[str]
```

### OpenRouter（主要提供商）

**位置**：`backend/app/provider/openrouter.py`

**特点**：
- 支持 100+ 模型
- 原生函数调用
- Reasoning token 支持
- 统一计费

**使用**：

```python
provider = OpenRouterProvider(api_key="sk-or-...")
async for chunk in provider.stream_chat(messages, "anthropic/claude-sonnet-4"):
    print(chunk.delta, end="")
```

## 工具调用适配

### 适配层架构

```
provider/tool_calling/
  ├── 原生函数调用（OpenAI/Anthropic 格式）
  ├── Prompt-based 回退（无原生工具调用支持时）
  └── Reasoning token 适配
```

### 原生函数调用

对于支持原生 function calling 的模型（如 GPT-4、Claude）：

```python
# provider 直接传递 tools 参数
response = await client.chat.completions.create(
    model=model,
    messages=messages,
    tools=tool_definitions,
    stream=True,
)
```

### Prompt-based 回退

对于不支持原生工具调用的模型：

```python
# 将工具定义注入到 system prompt
if tools and not self.supports_tool_calling(model):
    messages = self._inject_tools_to_prompt(messages, tools)
```

### Reasoning Token 支持

对于支持 thinking/reasoning 的模型：

```python
async for chunk in provider.stream_chat(messages, model):
    if chunk.reasoning:
        # 发送 reasoning_delta SSE 事件
        await job.send_event(REASONING_DELTA, {"text": chunk.reasoning})
    if chunk.delta:
        # 发送 text_delta SSE 事件
        await job.send_event(TEXT_DELTA, {"text": chunk.delta})
```

## 创建自定义提供商

### 步骤 1：继承 BaseProvider

创建 `backend/app/provider/my_provider.py`：

```python
from app.provider.base import BaseProvider, ChatMessage, StreamChunk, ModelInfo
from typing import AsyncGenerator, Optional
import httpx
import json


class MyProvider(BaseProvider):
    def __init__(self, api_key: str, base_url: str = "https://api.myprovider.com"):
        self.api_key = api_key
        self.base_url = base_url
        self.client = httpx.AsyncClient(
            timeout=httpx.Timeout(connect=10.0, read=120.0, write=10.0, pool=10.0)
        )

    async def stream_chat(
        self,
        messages: list[ChatMessage],
        model: str,
        **kwargs
    ) -> AsyncGenerator[StreamChunk, None]:
        api_messages = [
            {"role": m.role, "content": m.content if isinstance(m.content, str) else str(m.content)}
            for m in messages
        ]

        async with self.client.stream(
            "POST",
            f"{self.base_url}/v1/chat/completions",
            headers={
                "Authorization": f"Bearer {self.api_key}",
                "Content-Type": "application/json",
            },
            json={
                "model": model,
                "messages": api_messages,
                "stream": True,
                **kwargs
            }
        ) as response:
            async for line in response.aiter_lines():
                if line.startswith("data: "):
                    data = line[6:]
                    if data == "[DONE]":
                        yield StreamChunk(finish_reason="stop")
                        break
                    try:
                        chunk = json.loads(data)
                        delta = chunk.get("choices", [{}])[0].get("delta", {})
                        content = delta.get("content")
                        if content:
                            yield StreamChunk(delta=content)
                    except json.JSONDecodeError:
                        pass

    async def all_models(self) -> list[ModelInfo]:
        response = await self.client.get(
            f"{self.base_url}/v1/models",
            headers={"Authorization": f"Bearer {self.api_key}"}
        )
        data = response.json()
        return [
            ModelInfo(
                id=m["id"],
                name=m["id"],
                provider="my_provider",
                context_window=m.get("context_window", 4096),
                capabilities={"vision": False, "reasoning": False}
            )
            for m in data.get("data", [])
        ]

    def supports_tool_calling(self, model: str) -> bool:
        return False  # 或根据模型判断

    async def close(self):
        await self.client.aclose()
```

## 重试逻辑

**位置**：`backend/app/session/retry.py`

```python
# 可重试的错误类型
is_retryable(error) → bool
    # 429 Rate Limit, 500 Server Error, 503 Service Unavailable

# 认证错误

    # 401 Unauthorized, 403 Forbidden
```

## 常见问题

### 如何支持多模态？

在 ChatMessage 中使用多模态内容：

```python
ChatMessage(
    role="user",
    content=[
        {"type": "text", "text": "描述这张图片"},
        {"type": "image_url", "image_url": {"url": "data:image/png;base64,..."}}
    ]
)
```

### 如何处理流式断开？

实现断线重连：通过 SSE 的 `Last-Event-ID` 头恢复。

### 如何自定义模型参数？

通过 kwargs 传递：`temperature`, `max_tokens`, `top_p` 等。
