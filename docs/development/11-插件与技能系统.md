# OpenYak 插件与技能系统

## 概述

OpenYak 的插件系统（Cowork Plugin）是一个可扩展的领域专用功能包架构。每个插件可以包含：

- **Skills（技能）**：预定义的 prompt 模板 + 参考文档 + 脚本
- **Commands（命令）**：快捷指令
- **References（参考文档）**：领域知识库
- **Scripts（脚本）**：可执行的工具脚本

插件系统允许第三方开发者创建领域专用的功能包，用户可以安装、启用/禁用这些插件来扩展 OpenYak 的能力。

## 插件架构

### 整体结构

```
数据层: data/plugins/
  └── {plugin-id}/              # 已安装的插件
      ├── .claude-plugin/       # 插件元数据
      │   └── plugin.json       # 插件配置
      └── skills/               # 技能目录
          └── {skill-id}/
              ├── SKILL.md      # 技能定义
              ├── references/   # 参考文档
              └── scripts/      # 可执行脚本

代码层: backend/app/
  ├── plugin/                   # 插件管理器
  │   ├── manager.py            # PluginManager（发现/加载/启用/禁用）
  │   └── ...
  └── skill/                    # 技能注册表
      ├── registry.py           # SkillRegistry（注册/查询/执行）
      └── ...
```

### 插件与技能的关系

```
Plugin（插件）
  ├── 包含 1~N 个 Skill（技能）
  ├── 每个插件有独立的生命周期（安装/启用/禁用/卸载）
  └── 插件启用后，所有技能自动注册到 SkillRegistry

Skill（技能）
  ├── 独立的功能单元
  ├── 通过 skill 工具调用
  ├── 可引用参考文档和脚本
  └── 不依赖插件也可独立存在（内置技能）
```

## 技能（Skill）详解

### 技能定义文件 SKILL.md

每个技能通过 `SKILL.md` 文件定义，包含以下部分：

```markdown
---
name: code-review
description: Perform a thorough code review
trigger: /code-review
---

# Code Review Skill

## Instructions

You are performing a code review. Analyze the following code for:

1. **Correctness**: Logic errors, edge cases
2. **Security**: Vulnerabilities, injection risks
3. **Performance**: Inefficient algorithms, memory leaks
4. **Readability**: Naming, comments, structure
5. **Best Practices**: Language-specific patterns

## Steps

1. Read the code files
2. Identify issues by category
3. Rate severity (Critical/High/Medium/Low)
4. Suggest fixes with code examples

## References

See `references/` for language-specific review checklists.
```

### 技能目录结构

```
skills/
└── code-review/
    ├── SKILL.md              # 技能定义
    ├── references/           # 参考文档
    │   ├── python-checklist.md
    │   ├── javascript-checklist.md
    │   └── security-patterns.md
    └── scripts/              # 可执行脚本
        ├── analyze-complexity.py
        └── check-dependencies.sh
```

### 技能类型

1. **Prompt-only 技能**：只有 SKILL.md，通过 prompt 模板扩展 agent 能力
2. **Reference 技能**：包含参考文档，注入到上下文中
3. **Script 技能**：包含可执行脚本，通过 bash 工具执行
4. **Hybrid 技能**：组合以上三种

## 技能调用流程

### 通过 skill 工具调用

```
Agent 决定使用 skill
  ↓
tool_call: skill({"skill": "code-review", "args": {...}})
  ↓
SkillRegistry.lookup("code-review")
  ↓
加载 SKILL.md + references + scripts
  ↓
构建扩展 prompt（技能指令 + 参考文档内容）
  ↓
注入到当前对话上下文
  ↓
Agent 按照技能指令执行
  ↓
如果技能有脚本 → 通过 bash 工具执行
  ↓
返回执行结果
```

### 技能在 System Prompt 中的注入

```python
# session/system_prompt.py
def build_skill_section(skills: list[str]) -> str:
    """构建技能注入段落"""
    sections = []
    for skill_id in skills:
        skill = skill_registry.get(skill_id)
        if skill:
            sections.append(f"## Skill: {skill.name}\n{skill.instructions}")
            if skill.references:
                sections.append(f"### References\n{skill.references}")
    return "\n\n".join(sections)
```

## PluginManager API

### 发现插件

```python
from app.plugin.manager import PluginManager

manager = PluginManager()

# 扫描所有可用插件
plugins = manager.discover_plugins()
# → [{"id": "engineering", "name": "Engineering", ...}, ...]

# 获取插件详情
plugin = manager.get_plugin("engineering")
# → PluginInfo(id="engineering", skills=[...], enabled=True)
```

### 启用/禁用插件

```python
# 启用插件
manager.enable_plugin("engineering")

# 禁用插件
manager.disable_plugin("engineering")

# 获取已启用的插件
enabled = manager.get_enabled_plugins()
```

### 技能查询

```python
from app.skill.registry import SkillRegistry

registry = SkillRegistry()

# 列出所有可用技能
skills = registry.list_skills()

# 按插件过滤
eng_skills = registry.list_skills(plugin_id="engineering")

# 搜索技能
results = registry.search_skills("code review")
```

## 创建自定义插件

### 步骤 1：创建插件目录

```bash
mkdir -p data/plugins/my-plugin/.claude-plugin
mkdir -p data/plugins/my-plugin/skills/my-skill
```

### 步骤 2：创建 plugin.json

```json
// data/plugins/my-plugin/.claude-plugin/plugin.json
{
  "name": "my-plugin",
  "version": "1.0.0",
  "description": "My custom plugin for specific domain",
  "author": "Your Name",
  "skills": ["my-skill"]
}
```

### 步骤 3：创建技能

```markdown
<!-- data/plugins/my-plugin/skills/my-skill/SKILL.md -->
---
name: my-skill
description: My custom skill for specific task
trigger: /my-skill
---

# My Custom Skill

## Instructions

You are an expert in [domain]. When the user asks about [topic]:

1. First, analyze the [input]
2. Then, apply [method]
3. Finally, present results in [format]

## Guidelines

- Always [guideline 1]
- Never [guideline 2]
- When [condition], do [action]
```

### 步骤 4：添加参考文档（可选）

```markdown
<!-- data/plugins/my-plugin/skills/my-skill/references/domain-guide.md -->
# Domain Guide

This document provides domain-specific knowledge...
```

### 步骤 5：添加脚本（可选）

```python
# data/plugins/my-plugin/skills/my-skill/scripts/analyze.py
"""Analysis script for my-skill"""
import sys
import json

def main():
    input_data = json.loads(sys.stdin.read())
    # 分析逻辑
    result = {"analysis": "..."}
    print(json.dumps(result))

if __name__ == "__main__":
    main()
```

### 步骤 6：安装插件

```bash
# 通过 API 安装
POST /api/plugins/install
{
  "path": "/path/to/my-plugin"
}

# 或手动复制到 data/plugins/ 目录
cp -r my-plugin data/plugins/
```

### 步骤 7：启用插件

```bash
POST /api/plugins/my-plugin/enable
```

## 最佳实践

### 技能设计原则

1. **单一职责**：每个技能聚焦一个明确任务
2. **可组合**：技能之间可组合使用
3. **渐进增强**：先 prompt-only，再添加 references，最后 scripts
4. **参考文档精简**：只包含关键知识，避免过长的上下文

### SKILL.md 编写规范

- 清晰的触发条件（trigger）
- 结构化的执行步骤
- 明确的输入/输出格式
- 包含错误处理指导
- 提供示例

### 脚本安全

- 所有脚本通过 bash 工具执行，受权限引擎控制
- 脚本应处理异常和超时
- 避免删除操作或使用 `sudo`
- 输出使用结构化格式（JSON）

### 参考文档管理

- 使用 Markdown 格式
- 文件大小控制在合理范围（避免上下文溢出）
- 按主题拆分（一个主题一个文件）
- 定期更新以保持准确性

## 常见问题

### 如何创建跨插件的技能？

技能通常属于一个插件。如果需要跨插件共享，可以：
1. 将通用技能作为独立插件
2. 在 SKILL.md 中引用其他插件的技能

### 如何调试技能？

1. 先手动执行 skill 的 prompt，验证效果
2. 检查参考文档是否被正确注入
3. 如有脚本，先独立测试脚本

### 如何控制技能的上下文消耗？

1. 精简 SKILL.md 中的指令
2. 限制参考文档的长度
3. 只在需要时动态加载参考文档
