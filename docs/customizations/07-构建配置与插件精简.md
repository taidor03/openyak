# 07 - 构建配置与插件精简

> **定制编号**: XFLOW-005, XFLOW-006, XFLOW-016  
> **涉及范围**: Tauri 构建配置 + 连接器/插件精简 + 默认端口/语言配置

---

## 一、功能概述

1. **Tauri 构建配置同步**：构建命令、更新器、资源路径、签名
2. **连接器精简**：移除 Xflow 无关的 OAuth 连接器
3. **插件精简**：删除无关业务插件，保留工程相关
4. **默认端口与配置调整**

---

## 二、Tauri 构建配置同步

### 2.1 构建命令

**`desktop-tauri/src-tauri/tauri.conf.json`**：

```json
{
  "build": {
    "beforeBuildCommand": "npm --prefix .. run build:frontend && npm --prefix .. run build:backend"
  }
}
```

- 构建前自动编译前端和后端
- 使用 `--prefix ..` 从项目根目录执行

### 2.2 更新器配置

```json
{
  "bundle": {
    "createUpdaterArtifacts": true  // 从 false 改为 true
  }
}
```

启用自动更新产物生成。

### 2.3 资源路径

```json
{
  "bundle": {
    "resources": {
      "../../backend/dist/openyak-backend": "backend",  // 相对路径
      "../../backend/resources/nodejs": "nodejs"         // 相对路径
    }
  }
}
```

从绝对路径改为相对路径，确保团队协作时不同开发者机器上都能正常构建。

### 2.4 签名配置

```json
{
  "macOS": {
    "signingIdentity": "Developer ID Application: wangzhang wu (46KF5Z549N)"
  }
}
```

从 `null` 改为具体的开发者签名身份。

## 三、连接器精简

### 3.1 保留的连接器

**`backend/app/data/connectors.json`** 仅保留以下连接器：

| 连接器 | 类别 | 类型 | 说明 |
|--------|------|------|------|
| Slack | 通讯 | remote | 团队协作 |
| Notion | 知识库 | remote | 文档管理 |
| GitHub | 开发 | remote | 代码仓库 |
| Datadog | 监控 | remote | 可观测性 |
| Figma | 设计 | remote | 设计协作 |
| Canva | 设计 | remote | 创意工具 |
| BigQuery | 数据 | remote | 数据分析 |
| Google Workspace | 生产力 | local | Gmail/Calendar/Drive/Docs（有专用 OAuth 流程） |

> **偏差修正**：初版实施时错误删除了 `google-workspace`。该连接器虽为 local 类型，但有专门的 Google OAuth 认证流程（`backend/app/api/google_auth.py`），且在多个保留插件的 `.mcp.json` 中被引用，必须保留在 catalog 中。

### 3.2 移除的连接器类别

- CRM 类（Salesforce、HubSpot 等）
- 分析类（Google Analytics、Mixpanel、Amplitude 等）
- 营销类（Mailchimp、HubSpot Marketing 等）
- 销售类（Outreach、Salesloft 等）
- 法务类（Ironclad、DocuSign 等）
- 运营类（PagerDuty、Opsgenie 等）
- 知识库类（Confluence / Atlassian 等）
- 生物研究类
- 客户支持类（Intercom 等）
- 企业搜索类
- 项目管理类（Linear、Asana、Monday、ClickUp 等）
- 数据平台类（Snowflake、Databricks、Hex、Definite 等）
- 其他（Fireflies、SimilarWeb、Pendo、Ms365 等）

### 3.3 保留插件清理

保留的插件的 `.mcp.json` 同步清理，移除不在 catalog 中的 MCP 引用，消除"其它"分类中的游离连接器。

> **偏差修正**：初版实施时遗漏了此步骤，6 个保留插件的 `.mcp.json` 仍包含大量不在 catalog 中的连接器引用（如 `atlassian`、`linear`、`asana`、`monday`、`clickup`、`amplitude`、`snowflake`、`databricks`、`hex`、`intercom`、`fireflies`、`similarweb`、`pendo`、`ms365` 等），导致这些"游离"连接器以 `source=builtin` 注册并显示在插件页面的链接器 Tab 中。

修正后的各插件 `.mcp.json` 仅引用 catalog 中存在的连接器：

| 插件 | 保留的连接器引用 |
|------|----------------|
| engineering | slack, notion, github, datadog, google-workspace |
| design | slack, figma, canva, notion, google-workspace |
| productivity | slack, notion, bigquery, google-workspace |
| data | bigquery, datadog |
| human-resources | slack, google-workspace, notion |
| finance | bigquery, slack, google-workspace |
| product-management | slack, notion, figma, google-workspace |

## 五、默认端口与配置调整

- `backend/app/config.py`
- `backend/run.py`
- `frontend/src/lib/constants.ts`
