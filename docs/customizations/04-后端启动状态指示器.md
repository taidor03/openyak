# 04 - 后端启动状态指示器

> **定制编号**: XFLOW-014 + 后续重构  
> **涉及提交**: `d4384fe`, `6098791`(部分), `30c6d38`, `d32de62`, `f71e07c`  
> **涉及范围**: 前端指示器组件 + 后端健康探测 + Tauri IPC + Rust 后端

---

## 一、功能概述

桌面端冷启动时，前端 UI 从本地缓存立即显示，但后端尚在启动（加载提供商、插件、MCP 等需约 5-30s），用户无法感知就绪状态。本模块实现了完整的后端启动状态指示与通知机制，经过多轮重构优化。

### 演进历程

| 阶段 | 提交 | 方案 |
|------|------|------|
| V1 | `d4384fe` | `/startup-status` 轮询 + `useBackendReady` hook |
| V2 | `30c6d38` | Context Provider + `/livez` 轻量探测 |
| V3 | `d32de62` | Tauri IPC 事件驱动 + Rust backend_ready 状态位 |
| V4 | `f71e07c` | 移除 `/livez` 轮询降级，完全 IPC 事件驱动 |

### 3.3 /livez 探测

```
GET /livez
Response: 200 OK（无 body，极轻量）
```

### 3.4 Phase 流转

```
connecting → backend_ok → syncing → synced → done
```

- **connecting**：后端未就绪
- **backend_ok**：`/livez` 返回 200
- **syncing**：前端缓存同步中
- **synced**：缓存同步完成
- **done**：指示器隐藏

### 3.5 指示器简化

## 四、V3 — Tauri IPC 事件驱动（d32de62）

### 4.1 Rust 后端变更

**`desktop-tauri/src-tauri/src/backend.rs`**：

```rust
struct BackendInner {
    // ... 原有字段
    backend_ready: bool,  // 新增：后端就绪状态位
}

impl BackendState {
    /// IPC 命令：前端查询后端是否就绪
    pub async fn is_ready(&self) -> bool {
        let inner = self.inner.lock().await;
        inner.backend_ready
    }

    /// 标记后端就绪（开发模式下后端已运行时使用）
    pub async fn set_ready(&self) {
        let mut inner = self.inner.lock().await;
        inner.backend_ready = true;
    }

    /// 启动后端进程
    pub async fn start(&self, app: &AppHandle) -> Result<String, String> {
        // 1. 标记 backend_ready = false
        // 2. 启动后端进程
        // 3. 后端启动成功后 → backend_ready = true
        // 4. 发送 "backend-ready" 事件通知前端
    }
}
```

### 4.2 IPC 命令注册

**`desktop-tauri/src-tauri/src/commands.rs`**：

```rust
#[tauri::command]
async fn is_backend_ready(state: State<'_, BackendState>) -> bool {
    state.is_ready().await
}
```

### 4.3 窗口显示策略

**`desktop-tauri/src-tauri/src/lib.rs`**：

- **生产模式**：窗口立即显示，后端在后台启动
- 启动完成后通过 `backend-ready` 事件通知前端
- **开发模式**：后端已运行时立即标记 ready，跳过轮询

## 五、V4 — 完全事件驱动（f71e07c）

### 5.1 变更

- `BackendReadyProvider` 移除 `/livez` 轮询逻辑
- 完全依赖 Tauri IPC 事件驱动
- IPC 就绪检查失败时不再降级到 fetch 轮询
- 保持逻辑简洁

### 5.2 后端配合

**`backend/app/main.py`**：

MCP 连接移至后台任务后，HTTP 服务更快就绪，IPC 事件更早触发。
