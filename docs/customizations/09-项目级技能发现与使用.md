## 2. 后端定制

### 2.1 `_skill_source()` 增加 `agent` 来源识别

**文件**：`backend/app/api/skills.py`

```python
def _skill_source(skill_name: str, location: str) -> str:
    """Determine the source of a skill: 'plugin', 'bundled', 'agent', or 'project'."""
    if ":" in skill_name:
        return "plugin"
    if "/data/skills/" in location or "\\data\\skills\\" in location:
        return "bundled"
    # Skills from .agents/<agent-name>/skills/ are per-agent skills
    if "/.agents/" in location or "\\.agents\\" in location:
        return "agent"
    return "project"
```

`.agents/<agent-name>/skills/` 目录下的技能被识别为 `agent` 来源，与 `.openyak/skills/` 的 `project` 来源区分。

### 2.2 `list_skills` 增加 `workspace_path` 查询参数

**文件**：`backend/app/api/skills.py`

```python
@router.get("/skills")
async def list_skills(
    registry: SkillRegistryDep,
    workspace_path: str | None = None,
) -> list[dict[str, Any]]:
    """List all discovered skills.

    If *workspace_path* is provided and differs from the registry's
    current project directory, the registry is rescanned for that
    project so that per-project / per-agent skills are up to date.
    """
    if workspace_path:
        resolved = Path(workspace_path).resolve()
        if resolved.is_dir():
            registry_dir = Path(registry._project_dir).resolve() if registry._project_dir else None
            if registry_dir != resolved:
                registry.scan(project_dir=str(resolved))
    return [_skill_to_dict(skill, registry) for skill in registry.all_skills()]
```

### 2.4 新增技能 CRUD 端点

**文件**：`backend/app/api/skills.py`

| 端点 | 方法 | 功能 |
|------|------|------|
| `/api/skills/create` | POST | 创建项目级/Agent 级技能文件 |
| `/api/skills/{skill_name}` | PUT | 更新技能内容（保留 frontmatter） |
| `/api/skills/{skill_name}` | DELETE | 删除技能文件及目录 |

**CreateSkillRequest 参数**：

| 字段 | 类型 | 说明 |
|------|------|------|
| `name` | string | 技能名称 |
| `description` | string | 技能描述 |
| `content` | string | 技能正文（默认空） |
| `target` | string | 创建位置：`"project"` → `.openyak/skills/`，`"agents"` → `.agents/skills/`，`"<agent>"` → `.agents/<agent>/skills/` |
| `workspace_path` | string? | 当前工作区路径 |

**辅助函数**：

- `_resolve_project_dir(registry, workspace_path)` — 解析项目目录，优先使用前端传递的 workspace_path
- `_skill_target_dir(target, project_dir, slug)` — 根据目标类型计算技能文件路径

**关键逻辑**：

- 创建时自动生成 YAML frontmatter（`name` + `description`）
- 创建后自动 `registry.scan()` 刷新缓存
- 更新时保留原有 frontmatter，只替换正文
- 删除时递归移除技能目录（含附带的 scripts/references 等文件）
- 禁止编辑/删除 `bundled` 来源的技能（返回 403）

```typescript
export function useSkills(workspacePath?: string | null) {
  return useQuery({
    queryKey: [...queryKeys.skills, workspacePath ?? ""],
    queryFn: () => {
      const params = workspacePath
        ? `?workspace_path=${encodeURIComponent(workspacePath)}`
        : "";
      return api.get<SkillInfo[]>(`${API.SKILLS.LIST}${params}`);
    },
    staleTime: 30_000,
    refetchInterval: 60_000,
    meta: { persist: true },
  });
}
```

## 4. 数据流

```
用户打开项目 → activeWorkspacePath 更新
    → useSkills(workspacePath) → GET /api/skills?workspace_path=...
    → 后端 list_skills 对比 registry._project_dir
    → 不同则 registry.scan(project_dir=新路径)
    → SkillRegistry 递归扫描:
        .openyak/skills/**/SKILL.md     → source=project
        .agents/<name>/skills/SKILL.md  → source=agent
        .claude/skills/SKILL.md         → source=project (外部兼容)
    → 返回技能列表 → LLM 系统提示注入 render_skills_section()
    → LLM 调用 skill 工具 → 活动面板显示 ✨ Skill: xxx
```
