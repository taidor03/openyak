## 一、功能概述

本模块记录了桌面端冷启动体验优化的核心数据持久化策略，目标是"冷启动即显即刷"：

1. **TanStack Query 持久化**：通用查询缓存持久化到 localStorage
2. **会话列表本地缓存**：侧边栏即时渲染
3. **连接器/插件/技能即时缓存同步**：mutation 操作即时更新本地缓存，消除 invalidateQueries 闪烁

> **提示**：Provider/Model 本地缓存已独立至 [08-Provider与模型管理.md](./08-Provider与模型管理.md)。

## 二、TanStack Query 持久化（e471750）

### 2.1 依赖

### 2.2 持久化模块

**新增文件**：`frontend/src/lib/query-persister.ts`

选择性持久化层，仅持久化 `meta.persist=true` 的查询：

```typescript
import { QueryClient } from "@tanstack/react-query";
import { persistQueryClient } from "@tanstack/react-query-persist-client";
import { createSyncStoragePersister } from "@tanstack/query-sync-storage-persister";

const PERSIST_MAX_AGE_MS = 24 * 60 * 60 * 1000; // 24h

const persister = createSyncStoragePersister({
  storage: window.localStorage,
  key: "openyak_query_cache",
  serialize: JSON.stringify,
  deserialize: JSON.parse,
});

// 选择性持久化：仅 meta.persist=true 的查询
function persistFilter(query) {
  return query.meta?.persist === true;
}

export function setupQueryPersister(queryClient: QueryClient) {
  persistQueryClient({
    queryClient,
    persister,
    maxAge: PERSIST_MAX_AGE_MS,
    hydrateOptions: {},
    dehydrateOptions: {
      shouldDehydrateQuery: persistFilter,
    },
  });
}
```

### 2.3 Provider 集成

**`frontend/src/components/providers/query-provider.tsx`**：

```typescript
const [isRestoring] = useState(() => {
  setupQueryPersister(queryClient);
  return true;
});

// localStorage 可用时（非 SSR / 非隐私模式）
<PersistQueryClientProvider
  client={queryClient}
  persistOptions={{
    persister,
    maxAge: PERSIST_MAX_AGE_MS,
  }}
>
  {children}
</PersistQueryClientProvider>

// localStorage 不可用时优雅降级
<QueryClientProvider client={queryClient}>
  {children}
</QueryClientProvider>
```

### 2.5 过期策略

- `PERSIST_MAX_AGE_MS = 24h`：超过 24 小时的缓存数据在 hydration 时丢弃
- 持久化数据标记为 stale，触发后台刷新
- 实现"即显即刷"：冷启动从缓存渲染 → 后台刷新 → UI 静默更新

### 2.6 后端就绪后的缓存同步

**`frontend/src/components/providers/backend-ready-provider.tsx`**

桌面端启动时，后端就绪后立即 `invalidateQueries` 刷新所有结构数据：

```typescript
const STARTUP_SYNC_QUERY_KEYS = [
  queryKeys.connectors,
  queryKeys.mcpConfig,
  queryKeys.plugins.all,
  queryKeys.skills,
  queryKeys.agents,
  queryKeys.models,
  queryKeys.providers,
  queryKeys.tools,
] as const;

// 后端就绪后执行
await Promise.all(
  STARTUP_SYNC_QUERY_KEYS.map((queryKey) =>
    queryClient.invalidateQueries({ queryKey, refetchType: "all" }),
  ),
);
```

## 四、连接器/插件/技能即时缓存同步

### 4.1 问题

原始实现中，mutation 操作（启用/禁用连接器、插件、技能）成功后仅调用 `invalidateQueries`。这导致：
1. 缓存数据先被标记为 stale
2. UI 在重新拉取完成前显示旧数据或空白
3. 产生可见的闪烁/跳变

### 4.2 解决方案：即时缓存更新 + 后台刷新

mutation 成功后立即用 `setQueryData` 更新本地缓存，然后 `invalidateQueries` 后台刷新保持数据一致性：

```
mutation 成功
    ↓
setQueryData: 即时更新缓存 → UI 立即反映变化
    ↓
invalidateQueries: 后台刷新 → 确保与服务端一致
```

### 4.3 连接器即时缓存同步

**文件**：`frontend/src/hooks/use-connectors.ts`

新增两个缓存操作工具函数：

```typescript
/** 更新缓存中单个连接器的字段 */
function updateConnectorInCache(
  queryClient,
  connectorId: string,
  updater: (c: ConnectorInfo) => ConnectorInfo,
) {
  queryClient.setQueryData<ConnectorsResponse>(queryKeys.connectors, (old) => {
    if (!old) return old;
    const connector = old.connectors[connectorId];
    if (!connector) return old;
    return {
      ...old,
      connectors: {
        ...old.connectors,
        [connectorId]: updater(connector),
      },
    };
  });
}

/** 从缓存中移除一个连接器 */
function removeConnectorFromCache(queryClient, connectorId: string) {
  queryClient.setQueryData<ConnectorsResponse>(queryKeys.connectors, (old) => {
    if (!old) return old;
    const { [connectorId]: _, ...rest } = old.connectors;
    return { ...old, connectors: rest };
  });
}
```

**各 mutation 的即时更新逻辑**：

| Mutation | 即时更新 |
|----------|---------|
| `useConnectorToggle` | `enabled`/`status`/`connected` |
| `useConnectorDisconnect` | `connected=false`, `status="disconnected"` |
| `useConnectorReconnect` | `connected=true`, `status="connected"`, `error=null` |
| `useRemoveConnector` | 从 `connectors` 中移除该条目 |
| `useSetConnectorToken` | 仅 invalidate（token 不在缓存中） |
| `useAddCustomConnector` | 仅 invalidate（需完整后端响应） |

### 4.4 插件即时缓存同步

**文件**：`frontend/src/hooks/use-plugins.ts`

```typescript
/** 更新缓存中单个插件的 enabled 状态 */
function updatePluginInCache(queryClient, pluginName: string, enabled: boolean) {
  queryClient.setQueryData<PluginsStatusResponse>(queryKeys.plugins.all, (old) => {
    if (!old) return old;
    return {
      ...old,
      plugins: old.plugins.map((p) =>
        p.name === pluginName ? { ...p, enabled } : p,
      ),
    };
  });
}
```

| Mutation | 即时更新 |
|----------|---------|
| `usePluginToggle` | 插件 `enabled` 状态 + 技能列表 invalidate |

### 4.5 技能即时缓存同步

**文件**：`frontend/src/hooks/use-plugins.ts`

```typescript
/** 更新缓存中单个技能的 enabled 状态 */
function updateSkillInCache(queryClient, skillName: string, enabled: boolean) {
  queryClient.setQueryData<SkillInfo[]>(queryKeys.skills, (old) => {
    if (!old) return old;
    return old.map((s) =>
      s.name === skillName ? { ...s, enabled } : s,
    );
  });
}
```

| Mutation | 即时更新 |
|----------|---------|
| `useSkillToggle` | 技能 `enabled` 状态 |

### 4.6 MCP 配置即时缓存同步

**文件**：`frontend/src/hooks/use-mcp-config.ts`

```typescript
// useSaveMcpConfig onSuccess
onSuccess: (_data, mcpServers) => {
  // 立即用提交的数据更新缓存
  queryClient.setQueryData<McpUserConfig>(queryKeys.mcpConfig, {
    mcpServers,
  });
  // 后台刷新
  queryClient.invalidateQueries({ queryKey: queryKeys.mcpConfig });
  queryClient.invalidateQueries({ queryKey: queryKeys.connectors });
};
```

## 五、缓存架构总览

```
┌─────────────────────────────────────────────┐
│              localStorage                    │
├─────────────────────────────────────────────┤
│ TanStack Query Persist (24h TTL)            │
│  ├ agents / connectors / mcpConfig          │
│  └ plugins / skills                         │
├─────────────────────────────────────────────┤
│ Session Cache (版本化)                       │
│  └ sessions 全量快照                         │
└─────────────────────────────────────────────┘

写入流程: setQueryData(即时) → invalidateQueries(后台刷新)
读取流程: localStorage 缓存 → 即显 → 后台刷新 → UI 静默更新

注：Provider/Model 缓存已独立至 08-Provider与模型管理.md
```
