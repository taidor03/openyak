## ⚠ 实现偏差记录（2025-05 修复）

### 偏差1: 归档视图显示所有会话（含未归档）

**现象**: 点击归档按钮后，归档视图显示全部会话（已归档 + 未归档），而非仅显示已归档会话。

**根因**: `useArchivedSessions` hook 调用 `API.SESSIONS.LIST(PAGE_SIZE, pageParam, true)`，对应后端 `?include_archived=true`。后端 `list_sessions` 收到 `include_archived=True` 后仅跳过 `time_archived IS NULL` 过滤，返回的是**所有会话**（归档 + 未归档），而非仅归档会话。

**修复**: 后端 `list_sessions` 新增 `archived_only: bool = False` 参数。当 `archived_only=True` 时：
- 仅返回 `time_archived IS NOT NULL` 的会话
- 按 `time_archived DESC` 排序（最近归档的在前）

前端 `API.SESSIONS.LIST` 第三参数改为 `archivedOnly`，传 `?archived_only=true`。

```python
# backend/app/session/manager.py
if archived_only:
    stmt = stmt.where(Session.time_archived.isnot(None))
    stmt = stmt.order_by(Session.time_archived.desc())
elif not include_archived:
    stmt = stmt.where(Session.time_archived.is_(None))
    # ... 正常排序
```

```typescript
// frontend/src/lib/constants.ts
LIST: (limit = 50, offset = 0, archivedOnly = false) =>
  `/api/sessions?limit=${limit}&offset=${offset}${archivedOnly ? "&archived_only=true" : ""}`,
```

**涉及文件**: `backend/app/session/manager.py`、`frontend/src/lib/constants.ts`、`frontend/src/hooks/use-sessions.ts`

### 2.5 后端 API 支持

**`backend/app/session/manager.py`**：

`list_sessions` 函数新增 `include_archived: bool = False` 和 `archived_only: bool = False` 参数：

```python
async def list_sessions(
    limit: int = 50,
    offset: int = 0,
    project_id: str | None = None,
    include_archived: bool = False,
    archived_only: bool = False,
) -> list[Session]:
    stmt = select(Session).where(Session.parent_id.is_(None))
    if archived_only:
        stmt = stmt.where(Session.time_archived.isnot(None))
        stmt = stmt.order_by(Session.time_archived.desc())
    elif not include_archived:
        stmt = stmt.where(Session.time_archived.is_(None))
        stmt = stmt.order_by(
            Session.is_pinned.desc(),
            Session.time_created.desc(),
        )
    else:
        stmt = stmt.order_by(
            Session.is_pinned.desc(),
            Session.time_created.desc(),
        )
    stmt = stmt.offset(offset).limit(limit)
    # ...
```

> `time_archived desc` 排序，确保归档视图只展示归档内容。

### 2.6 useArchivedSessions Hook

**`frontend/src/hooks/use-sessions.ts`**：

```typescript
const PAGE_SIZE = 50;

export function useArchivedSessions(enabled: boolean) {
  return useInfiniteQuery({
    queryKey: ["sessions", "archived"] as const,
    queryFn: ({ pageParam = 0 }) =>
      api.get<SessionResponse[]>(API.SESSIONS.LIST(PAGE_SIZE, pageParam, true)),
    // API.SESSIONS.LIST(..., true) → ?archived_only=true
    initialPageParam: 0,
    enabled,
    getNextPageParam: (lastPage, _allPages, lastPageParam) => {
      if (lastPage.length < PAGE_SIZE) return undefined;
      return lastPageParam + PAGE_SIZE;
    },
  });
}
```

> **修复说明**：之前传 `include_archived=true` 会返回全部会话（含未归档），
> 导致归档视图显示所有会话。改为 `archived_only=true` 后仅返回已归档会话。

## 四、会话列表本地缓存（XFLOW-013）

### 4.1 缓存模块

**新增文件**：`frontend/src/lib/session-cache.ts`

```typescript
const SESSIONS_KEY = "xflow:sessions_v1";

interface CacheEntry<T> {
  data: T;
  updatedAt: number; // epoch ms
}

function readCache<T>(key: string): CacheEntry<T> | null {
  if (typeof window === "undefined") return null;
  try {
    const raw = localStorage.getItem(key);
    if (!raw) return null;
    return JSON.parse(raw) as CacheEntry<T>;
  } catch {
    return null;
  }
}

function writeCache<T>(key: string, data: T): void {
  if (typeof window === "undefined") return;
  try {
    localStorage.setItem(key, JSON.stringify({ data, updatedAt: Date.now() }));
  } catch {
    // quota exceeded or private-browsing — ignore silently
  }
}

export function readSessionsCache(): CacheEntry<SessionResponse[]> | null {
  return readCache<SessionResponse[]>(SESSIONS_KEY);
}

export function writeSessionsCache(data: SessionResponse[]): void {
  writeCache(SESSIONS_KEY, data);
}
```

## 五、活跃生成状态指示

### 5.1 activeSessionIds 组件局部轮询

**`frontend/src/components/layout/session-list.tsx`**：

轮询 `GET /chat/active` 获取当前正在生成的会话列表，`activeSessionIds` 作为组件局部 `useState` 管理（非全局 store）：

```typescript
const ACTIVE_POLL_INTERVAL = 5_000; // 5 seconds

const [activeSessionIds, setActiveSessionIds] = useState<Set<string>>(new Set());

useEffect(() => {
  let active = true;
  let timer: ReturnType<typeof setTimeout> | null = null;
  const poll = async () => {
    if (!active) return;
    try {
      const jobs = await api.get<Array<{ stream_id: string; session_id: string }>>(API.CHAT.ACTIVE);
      if (!active) return;
      const ids = new Set(jobs.map((j) => j.session_id));
      setActiveSessionIds(ids);
    } catch {
      // Silently ignore
    }
    if (active) {
      timer = setTimeout(poll, ACTIVE_POLL_INTERVAL);
    }
  };
  poll();
  return () => { active = false; if (timer) clearTimeout(timer); };
}, []);
```

## 六、会话切换不中止后端任务

### 6.1 设计决策

当用户导航离开正在生成的会话时：
- **不中止**后端生成任务（用户可以切回查看结果）
- **只清理**前端流式状态（UI 不卡在 "generating"）

### 6.2 实现

**`frontend/src/components/chat/chat-view.tsx`**：

在 sessionId 变化的 cleanup effect 中：

```typescript
// 只清理前端状态，不中止后端任务
// The SSE cleanup hook (use-sse.ts) already closes the client and
// schedules a delayed finishGeneration recovery.
state.finishGeneration();
```

### 6.3 SSE 清理恢复

**`frontend/src/hooks/use-sse.ts`**：

SSE 连接关闭时，如果 3 秒内没有新的 SSE hook 接管，强制 finishGeneration：

```typescript
setTimeout(() => {
  if (store.getState().isGenerating && store.getState().sessionId === capturedSessionId) {
    console.warn("SSE cleanup recovery: forcing finishGeneration after navigation away");
    store.getState().finishGeneration();
    connectionStore.getState().setStatus("idle");
  }
}, 3_000);
```
