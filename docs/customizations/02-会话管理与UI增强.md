## 一、功能概述

本模块涵盖会话管理相关的所有定制增强，包括：

1. **归档会话视图增强**（XFLOW-008）：归档切换按钮、已归档会话操作、归档后端 API
2. **项目行快速新建对话**（XFLOW-009）：项目行悬停新建按钮
3. **会话列表本地缓存**（XFLOW-013）：冷启动即时渲染
4. **活跃生成状态指示**：侧边栏脉冲指示器 + activeSessionIds 轮询
5. **会话切换不中止后端任务**：导航离开时只清理前端状态
6. **侧边栏导航增强**：内容工作台 + 知识中心入口

> ⚠️ **关于"会话拖拽排序"**：commit `38edce7` 的提交消息提到了"会话拖拽排序"，但**实际代码中并未实现** @dnd-kit 拖拽功能。该 commit 实际实现的是活跃生成状态指示和 SSE 增强。

## 五、活跃生成状态指示（38edce7 + e7eee89）

### 5.1 activeSessionIds 轮询

**`frontend/src/components/layout/session-list.tsx`**：

轮询 `GET /chat/active` 获取当前正在生成的会话列表：

```typescript
const activeSessionIds = useChatStore((s) => s.activeSessionIds);

useEffect(() => {
  let active = true;
  let timer: ReturnType<typeof setTimeout> | null = null;
  const poll = async () => {
    if (!active) return;
    try {
      const jobs = await api.get<Array<{ stream_id: string; session_id: string }>>(API.CHAT.ACTIVE);
      if (!active) return;
      const ids = new Set(jobs.map((j) => j.session_id));
      useChatStore.getState().setActiveSessionIds(ids);
    } catch {
      // Silently ignore
    }
    if (active) {
      timer = setTimeout(poll, ACTIVE_POLL_INTERVAL);
    }
  };
  poll();
  return () => { active = false; if (timer) clearTimeout(timer); };
}, []);
```

### 5.2 ChatStore 集成

**`frontend/src/stores/chat-store.ts`**：

```typescript
interface ChatStore {
  activeSessionIds: Set<string>;
  setActiveSessionIds: (ids: Set<string>) => void;
}

// setActiveSessionIds 优化：Skip update if set contents haven't changed
setActiveSessionIds: (ids) =>
  set((s) => {
    if (s.activeSessionIds.size === ids.size) {
      let same = true;
      for (const id of ids) {
        if (!s.activeSessionIds.has(id)) { same = false; break; }
      }
      if (same) return s; // no change — bail out
    }
    return { activeSessionIds: ids };
  }),
```

### 5.4 SSE 空闲恢复快速路径（e7eee89）

**`frontend/src/hooks/use-sse.ts`**：

```typescript
// Fast path: if the sidebar polling already knows this session has no
// active generation, force finishGeneration immediately — don't wait for
// the timestamp-based idle check, which can be defeated by stale heartbeats.
const state = store.getState();
if (state.sessionId && !state.activeSessionIds.has(state.sessionId)) {
  console.warn("SSE idle recovery: session no longer active, forcing finishGeneration");
  try {
    if (state.sessionId) {
      await finishFromDatabase(state.sessionId);
    }
  } finally {
    store.getState().finishGeneration();
    connectionStore.getState().setStatus("idle");
  }
  clearInterval(idleCheckTimer);
  client.close();
  return;
}
```

## 六、会话切换不中止后端任务（38edce7）

### 6.1 设计决策

当用户导航离开正在生成的会话时：
- **不中止**后端生成任务（用户可以切回查看结果）
- **只清理**前端流式状态（UI 不卡在 "generating"）

### 6.2 实现

**`frontend/src/components/chat/chat-view.tsx`**：

```typescript
// 修复前：离开会话时调用 stop() 中止后端任务
capturedStopRef();

// 修复后：只清理前端状态
state.finishGeneration();
```

### 6.3 SSE 清理恢复

**`frontend/src/hooks/use-sse.ts`**：

SSE 连接关闭时，如果 3 秒内没有新的 SSE hook 接管，强制 finishGeneration：

```typescript
setTimeout(() => {
  if (store.getState().isGenerating && store.getState().sessionId === capturedSessionId) {
    console.warn("SSE cleanup recovery: forcing finishGeneration after navigation away");
    store.getState().finishGeneration();
    connectionStore.getState().setStatus("idle");
  }
}, 3_000);
```
