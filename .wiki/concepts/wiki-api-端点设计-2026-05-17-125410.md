# Wiki API 端点设计

知识中心后端提供完整的 REST API，定义在 `backend/app/api/wiki.py`，所有端点前缀为 `/api/wiki`，支持全局 Wiki 和项目级 Wiki 双模式（通过 `?workspace=` 参数切换）。

---

## 端点总览

| 方法 | 端点 | 功能 |
|---|---|---|
| `GET` | `/api/wiki/status` | 获取 Wiki 初始化状态 |
| `POST` | `/api/wiki/initialize` | 初始化 Wiki 目录结构 |
| `GET` | `/api/wiki/pages` | 列出所有页面（支持 `?category=` 过滤） |
| `POST` | `/api/wiki/pages` | 创建新页面 |
| `GET` | `/api/wiki/pages/{id}` | 获取单个页面详情 |
| `PUT` | `/api/wiki/pages/{id}` | 完全替换页面内容 |
| `PATCH` | `/api/wiki/pages/{id}` | 部分更新页面字段 |
| `DELETE` | `/api/wiki/pages/{id}` | 删除页面 |
| `GET` | `/api/wiki/pages/{id}/related` | 获取引用当前页面的关联页面 |
| `GET` | `/api/wiki/pages/{id}/history` | 获取页面历史版本 |
| `POST` | `/api/wiki/search` | 全文搜索 |
| `POST` | `/api/wiki/merge` | 章节合并 |
| `GET` | `/api/wiki/categories` | 获取分类列表 |
| `GET` | `/api/wiki/validate` | 健康检查（断链/孤页/过时） |
| `GET` | `/api/wiki/recent` | 最近修改页面 |
| `GET` | `/api/wiki/index` | 获取 index.md 内容 |

---

## 通用参数

### Workspace 参数

所有端点支持可选的 `?workspace=<path>` 查询参数：

```
GET /api/wiki/status                    # 全局 Wiki
GET /api/wiki/status?workspace=/path/to/project  # 项目 Wiki
```

- 省略 `workspace` → 使用全局 Wiki 根（`~/.xflow/wiki/`）
- 提供 `workspace` → 使用项目 Wiki 根（`{workspace}/.wiki/`）

## 端点详情

### GET /api/wiki/status

获取 Wiki 的当前状态。

**响应：**
```json
{
  "initialized": true,
  "wiki_root": "/Users/roubin/.xflow/wiki",
  "total_pages": 42,
  "categories": {
    "entities": 10,
    "concepts": 8,
    "sources": 12,
    "synthesis": 5,
    "comparison": 4,
    "queries": 3
  },
  "has_index": true
}
```

---

### POST /api/wiki/initialize

幂等初始化 Wiki 目录结构。

**请求体：** 空

**逻辑：**
- 创建 6 个分类目录
- 创建 `index.md`（如果不存在）
- 已存在则直接返回，不覆盖

---

### GET /api/wiki/pages

列出所有页面。

**查询参数：**
| 参数 | 类型 | 说明 |
|---|---|---|
| `category` | `string` | 可选，按分类过滤 |

**响应：**
```json
[
  {
    "page_id": "openyak-mcp-自定义配置-2026-05-17",
    "title": "OpenYak MCP 自定义配置",
    "category": "concepts",
    "path": "concepts/openyak-mcp-自定义配置-2026-05-17-124051.md"
  }
]
```

---

### POST /api/wiki/pages

创建新页面。

**请求体：**
```json
{
  "title": "页面标题",
  "content": "Markdown 正文内容",
  "category": "entities",
  "force": false
}
```

| 字段 | 必填 | 说明 |
|---|---|---|
| `title` | 是 | 页面标题 |
| `content` | 是 | Markdown 正文 |
| `category` | 否 | 分类，默认 `entities` |
| `force` | 否 | `true`=覆写已存在页面；`false`=存在则报错 |

**响应（创建成功）：**
```json
{
  "page_id": "页面标题-2026-05-17-143052",
  "title": "页面标题",
  "path": "entities/页面标题-2026-05-17-143052.md"
}
```

**响应（已存在且 force=false）：**
```json
{
  "exists": true,
  "existing_path": "entities/页面标题-2026-05-17-143052.md",
  "message": "页面 '页面标题' 已存在于 entities"
}
```

---

### GET /api/wiki/pages/{id}

获取单个页面详情。

**响应：**
```json
{
  "page_id": "openyak-mcp-自定义配置-2026-05-17",
  "title": "OpenYak MCP 自定义配置",
  "category": "concepts",
  "content": "---\ntitle: \"OpenYak MCP 自定义配置\"\n...",
  "path": "concepts/openyak-mcp-自定义配置-2026-05-17-124051.md"
}
```

---

### PUT /api/wiki/pages/{id}

完全替换页面内容（包括 frontmatter）。

**请求体：**
```json
{
  "title": "新标题",
  "content": "---\ntitle: \"新标题\"\ncategory: concepts\n---\n新内容",
  "category": "concepts"
}
```

---

### PATCH /api/wiki/pages/{id}

部分更新页面字段（不覆写整个文件）。

**请求体：**
```json
{
  "title": "可选新标题",
  "content": "可选新内容",
  "category": "可选新分类",
  "related": ["关联页面ID"],
  "sources": ["来源标题"]
}
```

所有字段均为可选，仅更新提供的字段。

---

### DELETE /api/wiki/pages/{id}

删除页面并清理关联引用。

**响应：**
```json
{
  "deleted": true,
  "path": "concepts/openyak-mcp-自定义配置-2026-05-17-124051.md",
  "cleaned_references": 2
}
```

`cleaned_references` 表示从其他页面中清理了多少个已失效的 wikilink 引用。

---

### GET /api/wiki/pages/{id}/related

获取引用了当前页面的关联页面。

**响应：**
```json
[
  {
    "page_id": "其他页面-id",
    "title": "引用此页面的页面标题",
    "category": "entities",
    "snippet": "...包含 [[OpenYak MCP 自定义配置]] 的上下文..."
  }
]
```

---

### GET /api/wiki/pages/{id}/history

获取页面历史版本（当前为占位实现，返回空列表）。

---

### POST /api/wiki/search

全文搜索。

**请求体：**
```json
{
  "query": "MCP 配置",
  "max_results": 10
}
```

**响应：**
```json
{
  "results": [
    {
      "page_id": "openyak-mcp-自定义配置-2026-05-17",
      "title": "OpenYak MCP 自定义配置",
      "category": "concepts",
      "snippet": "...包含匹配关键词的上下文片段...",
      "title_match": true,
      "score": 0.85
    }
  ]
}
```

- `title_match`: `true` 表示标题命中
- `score`: 匹配分数（0~1），基于 token 频率

---

### POST /api/wiki/merge

将源页面的指定章节合并到目标页面。

**请求体：**
```json
{
  "title": "目标页面标题",
  "content": "源页面内容",
  "category": "concepts",
  "sections": ["## 新增章节", "### 补充内容"]
}
```

> ⚠️ **注意**：当前后端实现查找同名页面作为目标，`sections` 参数暂未被后端使用，而是将整个源页面内容追加到目标页面末尾。

---

### GET /api/wiki/categories

获取分类列表及页面数量。

**响应：**
```json
[
  { "name": "entities", "path": "entities", "count": 10 },
  { "name": "concepts", "path": "concepts", "count": 8 },
  { "name": "sources", "path": "sources", "count": 12 },
  { "name": "synthesis", "path": "synthesis", "count": 5 },
  { "name": "comparison", "path": "comparison", "count": 4 },
  { "name": "queries", "path": "queries", "count": 3 }
]
```

---

### GET /api/wiki/validate

健康检查，检测 Wiki 完整性问题。

**查询参数：**
| 参数 | 类型 | 说明 |
|---|---|---|
| `check_stale_days` | `int` | 超过该天数未更新视为过时，默认 30 |
| `check_empty_categories` | `bool` | 是否检查空分类，默认 `true` |

**响应：**
```json
{
  "orphaned": [
    { "page_id": "孤页-id", "title": "孤页标题", "reason": "no_inbound_links" }
  ],
  "broken_links": [
    { "page_id": "页面-id", "title": "页面标题", "broken_link": "失效的目标" }
  ],
  "stale_pages": [
    { "page_id": "页面-id", "title": "页面标题", "last_updated": "2026-01-01" }
  ],
  "empty_categories": ["空分类名"]
}
```

---

### GET /api/wiki/recent

获取最近修改的页面。

**查询参数：**
| 参数 | 类型 | 说明 |
|---|---|---|
| `limit` | `int` | 返回数量，默认 20 |

**响应：**
```json
[
  { "page_id": "...", "title": "...", "updated": "2026-05-17T08:15:00+00:00", "category": "..." }
]
```

---

### GET /api/wiki/index

获取 `index.md` 内容（Wiki 的入口索引页面）。

**响应：**
```json
{
  "content": "# Wiki Index\n\n...",
  "path": "index.md"
}
```

---

## 错误码

| HTTP 状态码 | 场景 |
|---|---|
| `200` | 成功 |
| `400` | 请求参数错误（如 force=false 但页面已存在） |
| `404` | 页面不存在 |
| `500` | 内部错误（文件系统、解析错误等） |
