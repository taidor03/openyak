# Wikilink 解析与文件名生成

知识中心使用两个核心工具模块处理页面标识和链接解析：

- `backend/app/wiki/filename.py` — Unicode 感知的文件名/ slug 生成器
- `backend/app/wiki/resolver.py` — Wikilink 语法解析与页面路径查找

---

## filename.py — 文件名生成

### 设计来源

从 TypeScript 项目 `nashsu/llm_wiki` 的 `src/lib/wiki-filename.ts` 移植到 Python，文件名格式完全对齐。

### 文件名格式

```
{slug}-{YYYY-MM-DD}-{HHMMSS}.md
```

示例：`openyak-mcp-自定义配置-2026-05-17-124051.md`

### Slug 生成规则

`make_query_slug(title)` 对标题进行以下处理：

| 步骤 | 操作 | 示例 |
|---|---|---|
| 1 | NFKC 规范化 | 全角字符转半角 |
| 2 | 去除首尾空白 | `" OpenYak "` → `"OpenYak"` |
| 3 | 空白 → 连字符 | `"OpenYak MCP 配置"` → `"OpenYak-MCP-配置"` |
| 4 | 移除非 Unicode 字母/数字/连字符 | 特殊符号被清除 |
| 5 | 移除下划线 | `foo_bar` → `foobar` |
| 6 | 压缩连续连字符 | `a--b` → `a-b` |
| 7 | 移除首尾连字符 | `-abc-` → `abc` |
| 8 | 转小写 | `OpenYak` → `openyak` |
| 9 | 截断至 50 字符 | 超长标题被截断 |

> 对于 CJK 等无大小写区分的语言，第 8 步是 no-op，这些语言的 slug 保持原字符。

### 完整文件名生成

```python
from app.wiki.filename import make_query_filename
from datetime import datetime

# 自动使用当前 UTC 时间
info = make_query_filename("OpenYak MCP 自定义配置")
print(info.slug)    # "openyak-mcp-自定义配置"
print(info.date)    # "2026-05-17"
print(info.time)    # "124051"
print(info.filename) # "openyak-mcp-自定义配置-2026-05-17-124051.md"

# 可注入时间进行确定性测试
info = make_query_filename(
    "测试标题",
    now=datetime(2026, 5, 17, 12, 0, 0, tzinfo=timezone.utc)
)
print(info.filename)  # "ce-shi-biao-ti-2026-05-17-120000.md"
```

### FilenameInfo 数据结构

```python
@dataclass(frozen=True)
class FilenameInfo:
    slug: str       # openyak-mcp-自定义配置
    date: str       # 2026-05-17
    time: str       # 124051 (HHMMSS，无冒号)
    filename: str   # 完整文件名
```

### CJK 支持验证

| 输入 | slug 输出 |
|---|---|
| `"OpenYak MCP 配置"` | `"openyak-mcp-配置"` |
| `"默会知识 概述"` | `"默会知识-概述"` |
| `"XFlow 架构设计"` | `"xflow-架构设计"` |
| `"Get Started Guide"` | `"get-started-guide"` |
| `"  hello   world  "` | `"hello-world"` |

## resolver.py — Wikilink 解析与路径查找

### Wikilink 语法支持

```
[[页面标题]]           → 普通链接
[[目标|别名]]          → 自定义显示文本
```

### 核心数据类型

```python
@dataclass(frozen=True)
class WikilinkInfo:
    slug: str     # 链接目标（页面 slug）
    label: str    # 显示文本
```

### unwrap_wikilink(s)

解析 Wikilink 语法，剥离 `[[ ]]` 包裹：

```python
unwrap_wikilink("[[OpenYak MCP 配置]]")
# → WikilinkInfo(slug="OpenYak MCP 配置", label="OpenYak MCP 配置")

unwrap_wikilink("[[target|alias]]")
# → WikilinkInfo(slug="target", label="alias")

unwrap_wikilink("[[|alias]]")
# → WikilinkInfo(slug="alias", label="alias")  # 空 target，fallback 到 alias

unwrap_wikilink("普通文本")
# → WikilinkInfo(slug="普通文本", label="普通文本")  # 非 wikilink，原样返回
```

### resolve_wiki_page(wiki_root, slug)

在指定 Wiki 根目录下按 slug 查找页面文件，支持两级匹配：

**第一级：精确文件名匹配**

```python
resolve_wiki_page("/Users/roubin/.xflow/wiki", "openyak-mcp-配置")
# → "concepts/openyak-mcp-配置-2026-05-17-124051.md"  (如果该文件名存在)
```

**第二级：按 stem 匹配（去除日期后缀）**

```python
# 用户输入 slug 不带日期后缀，实际文件名带时间戳
# 统一转小写 + 空格→连字符，去掉文件名的 -YYYY-MM-DD-HHMMSS 后缀后比较
resolve_wiki_page("/Users/roubin/.xflow/wiki", "OpenYak MCP 配置")
# → 遍历所有分类目录，strip_date_suffix("openyak-mcp-配置-2026-05-17-124051")
#   = "openyak-mcp-配置"  → 匹配成功
```

### resolve_related_slug(wiki_root, ref)

解析 `related:` 字段引用，支持三种格式：

| 格式 | 示例 | 解析逻辑 |
|---|---|---|
| 路径引用 | `wiki/entities/dpao.md` | 相对于项目根解析绝对路径 |
| 文件名 | `dpao.md` | 在所有分类目录中查找 |
| Slug | `dpao` | 等同于 `dpao.md` |

```python
resolve_related_slug(wiki_root, "wiki/entities/dpao.md")
# → "/Users/roubin/gitdata/xflow-desktop/.wiki/entities/dpao.md"

resolve_related_slug(wiki_root, "openyak-mcp-自定义配置")
# → 在分类目录中查找 → "concepts/openyak-mcp-自定义配置-xxx.md"
```

### _strip_date_suffix(stem)

内部工具函数，去除文件名 stem 末尾的时间戳模式：

```python
_strip_date_suffix("openyak-mcp-配置-2026-05-17-124051")
# → "openyak-mcp-配置"

_strip_date_suffix("readme")   # 无时间戳
# → "readme"
```

### 前端 Wikilink 预处理

前端 `content.tsx` 中同样实现了 Wikilink 预处理：

```typescript
// [[target|alias]] → [alias](wiki:target)
function preprocessWikiMarkdown(raw: string): string {
  const content = raw.replace(/^---\n[\s\S]*?---\n/, "");
  return content.replace(
    /\[\[([^\]|]+?)(?:\|([^\]]+))?\]\]/g,
    (_, target, alias) => `[${alias || target}](wiki:${target.trim()})`
  );
}
```

### 前后端对比

| 层面 | 实现方式 |
|---|---|
| 后端（Python） | `resolver.py`：`unwrap_wikilink` / `resolve_wiki_page` / `resolve_related_slug` |
| 前端（TypeScript） | 正则预处理：`[[target\|alias]]` → `[alias](wiki:target)` |
| 前端渲染 | ReactMarkdown 自定义 `a` 组件：`wiki:` scheme 触发 `onWikilinkClick` |
| 跨端一致性 | 两者均使用相同的 Wikilink 语法 `[[ ]]`，但后端用于路径查找，前端用于 UI 渲染 |
