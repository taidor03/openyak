# WikiService 核心服务

`WikiService` 是知识中心的后端核心业务逻辑层，定义在 `backend/app/wiki/service.py`。它是一个**无状态**的服务类，所有方法都是 `@staticmethod`，每个方法都显式接收 `wiki_root` 参数，从而支持全局 Wiki 和项目级 Wiki 的动态切换。

---

## 初始化与路径

```python
from app.wiki import WikiService

# 解析 Wiki 根路径（全局或项目级）
wiki_root = WikiService.resolve_wiki_root(workspace=None)  # 全局: ~/.xflow/wiki/
wiki_root = WikiService.resolve_wiki_root(workspace="/path/to/project")  # 项目: /path/to/project/.wiki/
```

`resolve_wiki_root` 逻辑：
- `workspace` 为 `None` 或 `"."` → 使用 `~/.xflow/wiki/`
- `workspace` 有值 → 使用 `{workspace}/.wiki/`
- 无法确定 home 目录 → 返回 `None`

---

## 初始化 Wiki

```python
WikiService.initialize(wiki_root)
```

- **幂等性**：如果目录结构已存在则直接返回，不会覆盖已有内容
- 创建 6 个分类目录：`entities/`, `concepts/`, `sources/`, `synthesis/`, `comparison/`, `queries/`
- 创建 `index.md`（如果不存在），内容为默认知识索引模板
- 返回 `{"wiki_root", "initialized", "created_dirs", "created_files"}`

## 页面 CRUD

### 读取页面

```python
page = await WikiService.read_page(wiki_root, "openyak-mcp-自定义配置-2026-05-17")
# 返回: {"page_id": "...", "title": "OpenYak MCP 自定义配置", "content": "...", "category": "concepts", "path": "..."}
# 页面不存在 → None
```

- 从 YAML frontmatter 提取 `title`；无 frontmatter 则用文件名 stem 推断
- 提取分类从相对路径的第一级目录名获取

### 查找页面（按标题）

```python
page = await WikiService.find_page_by_title(wiki_root, "OpenYak MCP 配置", category="concepts")
# 返回同 read_page 格式的 dict，或 None
```

- 在指定分类目录下按 **规范化标题** 匹配（`normalize_wiki_ref_key` 比较）
- 仅读取文件头部 1 KB 来提取 frontmatter，避免读取大型页面全文

### 创建/更新页面

```python
result = await WikiService.write_page(
    wiki_root=wiki_root,
    title="新页面",
    content="页面正文内容",  # 不含 frontmatter，系统自动生成
    category="entities",     # 默认 "entities"
    force=False,             # False=存在则拒绝；True=覆写
)
```

- **自动生成 YAML frontmatter**：`title`, `category`, `created`, `updated`, `related`, `sources`
- `force=False` 时遇到**同分类下规范化标题重名**的页面：
  - 返回 `{"exists": True, "existing_content_preview": "...", "message": "..."}`
  - 提供已有内容的前 2000 字符预览，供 LLM 决定是否合并
- `force=True` 直接覆写文件，**保留已有的 `related` 和 `sources` 字段**
- 新页面自动使用 `make_query_filename` 生成文件名（`{slug}-{YYYY-MM-DD}-{HHMMSS}.md`）
- 写入后自动更新 `index.md`（添加 `[[title]] (category)` 条目）
- 写入后自动追加 `log.md` 记录

### 删除页面

```python
await WikiService.delete_page(wiki_root, page_id)
# 返回: True (成功) / False (页面不存在或删除失败)
```

- 删除对应 `.md` 文件
- 调用 `cleanup.build_deleted_keys()` 生成删除键集合
- 调用 `cleanup.clean_index_listing()` 清理 `index.md` 中指向该页面的列表项
- 调用 `cleanup.strip_deleted_wikilinks()` 清理所有页面中指向该页面的已失效 Wikilink

## 搜索

```python
results = await WikiService.search(
    wiki_root=wiki_root,
    query="MCP 配置",
    max_results=20,  # 默认 20
)
# 返回: list[SearchResult]
```

- 委托给 `search_wiki()` 执行
- 基于 **Token + 短语匹配**（非向量搜索），支持 CJK bigram 分词
- 评分权重：文件名精确(200) > 标题短语(50) > 内容短语(20/次) > 标题token(5) > 内容token(1)

---

## 页面合并

```python
result = await WikiService.merge_page(
    wiki_root=wiki_root,
    title="目标页面标题",
    new_sections="## 新章节\n新内容\n## 已有章节（更新版）\n更新内容",
    category="entities",  # 默认 "entities"
)
```

- 先查找同分类下同标题的已有页面
- **无已有页面** → 回退到 `write_page`（创建新页面）
- **有已有页面** → 调用 `cleanup.merge_sections()` 进行章节级合并：
  - 前言（第一个 `##` 之前的内容）始终保留
  - 匹配的章节：替换内容，保留原始标题级别
  - 未匹配的章节：追加到末尾
- 保留已有 frontmatter 的 `related` 和 `sources` 字段
- 写入后自动更新 `index.md` 和 `log.md`

## 来源吸收（Ingest）

```python
result = await WikiService.ingest_source(
    wiki_root=wiki_root,
    source_name="参考文档标题",
    source_content="文档正文内容...",
    purpose="general",  # 用途描述，默认 "general"
)
```

- 自动写入 `sources` 分类
- 创建结构化的源文档页面（含 Source、Content、Key Entities、Key Concepts、Related Pages 章节模板）
- 使用 `force=True` 覆写同名页面
- Wiki 目录不存在时自动初始化
- `log.md` 中记录动作为 `"ingest"`

## 健康检查（Lint）

```python
issues = await WikiService.lint_wiki(
    wiki_root=wiki_root,
    scope="full",  # 当前仅支持 "full"
)
# 返回: {"issues": [...], "total_issues": N, "warnings": N, "healthy": bool, "pages_checked": N, "summary": {...}}
```

检测项目：

| 类型 | 严重度 | 说明 |
|---|---|---|
| `orphan` | info | 孤页——无其他页面通过 `[[wikilink]]` 引用它 |
| `broken_wikilink` | warning | 失效的 Wikilink——指向不存在的页面 |
| `stale` | info | 超过 30 天未更新的页面 |
| `empty_category` | info | 空分类目录 |

## 技术要点

| 要点 | 说明 |
|---|---|
| **无状态设计** | 所有方法为 `@staticmethod`，不缓存文件列表，每次操作从磁盘读取 |
| **YAML frontmatter** | 用 `---` 包裹的元数据，支持 `title`, `category`, `created`, `updated`, `related`, `sources` 字段 |
| **幂等初始化** | `initialize()` 不会覆盖已有文件 |
| **Wikilink 解析** | 通过 `resolver.py` 解析 `[[target\|alias]]` 语法 |
| **防并发写** | 暂未实现文件锁，建议操作时避免并发写入同一 Wiki |
| **标题规范化** | 使用 `normalize_wiki_ref_key` 进行大小写、空格、连字符、下划线无关的比较 |
