# OpenYak MCP 自定义配置

## 配置文件位置

OpenYak 的 MCP 服务器配置存放在用户数据目录的 `.openyak/` 目录下：

| 文件 | 用途 |
|---|---|
| `.openyak/mcp-servers.json` | MCP 服务器主配置 |
| `.openyak/mcp-tokens.json` | 敏感令牌（如 API Key） |
| `.openyak/connectors.json` | 已启用的 Connector 列表 |

### 各平台路径

| 平台 | 路径 |
|---|---|
| **macOS** | `~/Library/Application Support/com.openyak.desktop/data/.openyak/mcp-servers.json` |
| **Windows** | `%APPDATA%\com.openyak.desktop\data\.openyak\mcp-servers.json` |
| **Linux** | `~/.config/com.openyak.desktop/data/.openyak/mcp-servers.json` |

> ⚠️ 注意：项目根目录下的 `.openyak/mcp-servers.json` 如果存在，会被**忽略**。请使用上述用户数据目录中的配置文件。

---

## mcp-servers.json 配置格式

```json
{
  "mcpServers": {
    "<server-name>": {
      "type": "remote | local",
      "url": "https://example.com/mcp",
      "headers": {
        "Authorization": "Bearer <token>"
      },
      "command": ["npx", "-y", "package-name"],
      "name": "显示名称",
      "description": "描述",
      "category": "search | tools | 自定义分类"
    }
  }
}
```

### 字段说明

| 字段 | 必填 | 说明 |
|---|---|---|
| `type` | 是 | `remote`（HTTP 远程）或 `local`（本地命令） |
| `url` | remote 必填 | MCP Server 的 HTTPS 端点 |
| `headers` | remote 可选 | 请求头，用于传递认证信息 |
| `command` | local 必填 | 启动命令数组，如 `["npx", "-y", "package-name"]` |
| `name` | 否 | 显示名称 |
| `description` | 否 | 描述信息 |
| `category` | 否 | 分类标签（如 `search`、`tools`） |

## 用户自定义配置示例

以下为 `mcp-servers.json` 中用户自行添加的配置示例（非 OpenYak 内置）：

```json
{
  "mcpServers": {
    "web-search": {
      "type": "remote",
      "url": "https://api.stepfun.com/step_plan/v1/mcp/web_search/mcp",
      "headers": {
        "Authorization": "Bearer 1GOUe6rM7iVf1WvSeoYkn917X3HlJgQw1ZA42sttL7KpBYscnX0euPogLJNcruHXa"
      },
      "name": "StepSearch Web Search",
      "description": "阶跃星辰 StepSearch 网页搜索 MCP",
      "category": "search"
    },
    "superpowers": {
      "type": "local",
      "command": ["npx", "-y", "superpowers-mcp"],
      "name": "superpowers",
      "description": "superpowers MCP",
      "category": "tools"
    }
  }
}
```

**注意**：OpenYak **本身没有全局默认的内置 MCP 配置**。`.openyak/mcp-servers.json` 中的内容完全是用户自定义添加的。

## 常见 MCP 配置模板

### Context7（库文档查询）

```json
"context7": {
  "type": "local",
  "command": ["npx", "-y", "@upstash/context7-mcp@latest"],
  "name": "Context7",
  "description": "库文档查询 MCP",
  "category": "docs"
}
```

### grep.app（代码搜索）

grep.app 需要先 `git clone` + `npm install` + `npm run build`，不能直接 `npx`：

```json
"grep_app": {
  "type": "local",
  "command": ["node", "/绝对路径/grep_app_mcp/dist/server-stdio.js"],
  "name": "Grep App",
  "description": "跨 GitHub 仓库代码搜索",
  "category": "search"
}
```

### StepFun Web Search（远程 MCP）

```json
"step-search": {
  "type": "remote",
  "url": "https://api.stepfun.com/step_plan/v1/mcp/web_search/mcp",
  "headers": {
    "Authorization": "Bearer ${STEPFUN_API_KEY}"
  },
  "name": "Step Search",
  "category": "search"
}
```

### 本地 Python MCP（uvx）

```json
"local-python-mcp": {
  "type": "local",
  "command": ["uvx", "package-name"],
  "name": "Python MCP",
  "category": "tools"
}
```

## 敏感信息管理

不要把 API Key 写死在 `mcp-servers.json` 里。应该在 `mcp-tokens.json` 里存储 token，在 `mcp-servers.json` 里用引用方式使用。

---

## connectors.json 与 mcp-servers.json 的区别

| 文件 | 作用 |
|---|---|
| `connectors.json` | 声明哪些 Connector 已启用（列表） |
| `mcp-servers.json` | 定义 MCP 服务器的实际连接配置 |

注意：`connectors.json` 里的 `grep-app` 当前只有启用标记，**没有对应的 MCP 配置**，需要手动在 `mcp-servers.json` 中添加。
