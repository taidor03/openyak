# 知识中心（Knowledge Center）架构总览

知识中心是 OpenYak 内置的 Wiki 知识管理系统，支持全局 Wiki 和项目级 Wiki 双模式，提供页面的 CRUD、全文搜索、章节合并、关联引用等核心功能。

---

## 整体架构

```
┌─────────────────────────────────────────────────────────┐
```

## 目录结构

```
# 后端
backend/app/
├── api/
│   └── wiki.py              # FastAPI 路由端点 (14 个)
└── wiki/
    ├── __init__.py          # WikiService 导出
    ├── service.py           # 核心业务逻辑
    ├── cleanup.py           # 索引清理与维护工具
    ├── search.py            # 基于 token 的全文搜索引擎
    ├── resolver.py          # Wikilink 解析与页面路径解析
    └── filename.py          # Unicode 感知的文件名生成

# 前端
frontend/src/app/(main)/
└── knowledge/
    ├── page.tsx             # 页面入口组件
    └── content.tsx          # 核心功能组件 (≈680 行)
```

## 双 Wiki 根策略

知识中心支持两种 Wiki 根路径，可通过前端切换：

| 模式 | Wiki 根路径 | 说明 |
|---|---|---|
| **全局 Wiki** | `~/.xflow/wiki/` | 跨项目共享，用户级配置 |
| **项目 Wiki** | `{workspace}/.wiki/` | 与代码库绑定的本地 Wiki |

后端通过 `?workspace=<path>` 参数动态确定使用哪个根目录，`WikiService.resolve_wiki_root()` 实现切换逻辑。

---

## 6 个默认分类

所有 Wiki 页面按以下分类目录组织（均在 Wiki 根下）：

| 分类 | 图标 | 用途 |
|---|---|---|
| `entities` | BookOpen | 实体：系统、模块、配置项等 |
| `concepts` | FileText | 概念：核心概念、设计理念等 |
| `sources` | FolderOpen | 来源：参考文档、第三方内容 |
| `synthesis` | GitMerge | 综合：整合性分析、对比总结 |
| `comparison` | Code2 | 比较：同类技术/方案的横向对比 |
| `queries` | Search | 查询：常见问题、FAQ |

## 核心功能矩阵

| 功能 | 前端 | 后端 | 存储 |
|---|---|---|---|
| 页面列表 | ✅ | ✅ `GET /pages` | 文件系统扫描 |
| 创建页面 | ✅ | ✅ `POST /pages` | 写入 `.md` 文件 |
| 读取页面 | ✅ | ✅ `GET /pages/{id}` | 读取 `.md` 文件 |
| 编辑页面 | ✅ | ✅ `PUT/PATCH /pages/{id}` | 覆写 `.md` 文件 |
| 删除页面 | ✅ | ✅ `DELETE /pages/{id}` | 删除文件 + 清理索引 |
| 全文搜索 | ✅ | ✅ `POST /search` | 内存 Token 匹配 |
| 章节合并 | ✅ | ✅ `POST /merge` | Markdown 章节合并 |
| 关联引用 | ✅ | ✅ `GET /pages/{id}/related` | Wikilink 反向查找 |
| 初始化 Wiki | ✅ | ✅ `POST /initialize` | 创建目录结构 |
| 健康检查 | ✅ | ✅ `GET /validate` | 断链/孤页/过时检测 |
| 最近修改 | ✅ | ✅ `GET /recent` | 按更新时间排序 |
| 分类管理 | ✅ | ✅ `GET /categories` | 扫描分类目录 |

## Wiki 页面格式

每个 Wiki 页面是标准 Markdown 文件，带有 YAML frontmatter：

```yaml
---
title: "页面标题"
category: entities
created: 2026-04-23T10:30:00+00:00
updated: 2026-05-17T08:15:00+00:00
related:
  - "其他页面 slug"
sources:
  - "来源标题"
---
```

文件名格式：`{slug}-{YYYY-MM-DD}-{HHMMSS}.md`

## Wikilink 系统

知识中心支持类似 Obsidian 的 Wikilink 语法：

```
[[页面标题]]           → 普通链接
[[目标|别名]]          → 自定义显示文本
[[related:页面ID]]     → 关联引用标记
```

前端 `WikiMarkdown` 组件在渲染时预处理 Wikilink，将其转换为 `wiki:` href scheme 供前端路由捕获。
